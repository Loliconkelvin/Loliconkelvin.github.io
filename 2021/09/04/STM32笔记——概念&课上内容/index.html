ARM处理器分类——按应用特征分类

  • ARM A系列 （application）

    偏应用型、高性能、合理功耗

  • ARM R系列 （real-time）

    注重实时性、较低功耗

  • ARM M系列 （micro）

    注重低功耗、低成本

GPIO（Kinetis KL25）（基于寄存器）

  • 点灯总攻略
    1. 配置时钟
    2. IO口模式选择
    3. PDDR（port degital direction register）IO方向选择（输入？输出？）
    4. PDIR（port degital input reg）输入控制
      或
      PDOR（port degital output reg）输出控制
  1. 时钟配置（SIM）

  • SIM_SCGC5的9-13位分别控制A-E五组引脚的时钟开关，为它们置一以开启他们的时钟
/*使用或运算将某一位置一，左移操作控制将哪一位置一
语义清除，比直接写16进制数更容易理解
如果是需要将两位置一的话，或许可以将0x03左移？
*/
SIM_SCGC5 |= (1 << n);
  1. IO口模式选择（MUX）
  • MUX寄存器（8-10位）控制指定引脚的工作模式（作为IO口还是用作其他功能？）
  • 因为一句只能控制一个引脚，所以往往需要写很多句
    /*
x为引脚组编号（A-E）
n为该组编号的引脚号（0-15）
*/

/*
将指定组的指定引脚控制为GPIO模式
即将10-8位置为001，那么11-0位就是
x001 xxxx xxxx。
只需要将原值与0001 0000 0000求与即可
16进制就是0x100
*/
PORTx_PCRn |= 0x100;
  1. 方向选择（PDDR）
  • PDDR寄存器的0-31号引脚分别控制指定组的32个引脚的方向
    置0则指定引脚为输入模式
    置1则指定引脚为输出模式
/*x为引脚组编号（A-E）
将该组0-7号引脚设置为输出模式
*/
GPIOx_PDDR |= 0xFF;
  1. 输出（PDOR）
  • 当引脚为GPIO输出模式时，PDOR控制指定组的32个引脚的逻辑输出电平
/*x为引脚组编号（A-E）
设置该组0-7号引脚输出低电平
*/
GPIOx_GPIO &= (~0xFF);
  1. 点灯总程序
  • 将D组0-7连接的LED灯点亮
int mian(void){
    //开启时钟
    SIM_SCGC5 |= (1 << 12);

    //配置为GPIO模式
    PORTD_PCR0 |= 0x100;
    PORTD_PCR1 |= 0x100;
    PORTD_PCR2 |= 0x100;
    PORTD_PCR3 |= 0x100;
    PORTD_PCR4 |= 0x100;
    PORTD_PCR5 |= 0x100;
    PORTD_PCR6 |= 0x100;
    PORTD_PCR7 |= 0x100;
    
    //配置为输出模式
    GPIOx_PDDR |= 0xFF;

    //配置为输出低电平
    GPIOx_GPIO &= (~0xFF);

    while(1);
}

嵌入式系统中的C语言

数据类型

  • 单片机编程独有代码规范：
    • 尽量使用满足要求的最小数据类型
    • 避免浮点数
    • 在不同系统，不同编译环境给每个数据类型重命名一个宽度值作为标记（单片机独有——移植性）
//For 8 Bit Machine
typedef unsigned char uint8_t;//8位 1字节数据类型
typedef int int16_t;//16位 2字节数据类型
typedef unsigned long uint32_t;//32位 4字节数据类型

//For 32 Bit Machine
typedef unsigned char uint8_t;//8位 1字节数据类型
typedef short int16_t;//16位 2字节数据类型
typedef unsigned int uint32_t;//32位 4字节数据类型

//注意，上面两段根据机器位数取一段使用
  • 在定义变量时使用重命名好的数据类型名（数据类型没有颜色，莫得灵魂）

GPIO（STM32F103CXXXX）（基于寄存器）

  • 还是那四步
    1. 开启时钟
    2. 配置IO口工作模式
    3. 配置IO口方向
    4. 写输出电平
  1. 开启时钟（RCC_APB2ENR）
  • 通过配置RCC_APB2ENR的第2-6位（分别对应IO口组A-G）以开启对应组的时钟
  1. 配置IO口方向与工作方式（GPIO_CPL/GPIO_CPH）
  • F103的IO口工作方向和工作方式是一起配置的，由于一组IO口有16个，因此使用两个寄存器，端口配置低寄存器（GPIOx_CPL） 和端口配置高寄存器（GPIOx_CPH） 分别用于控制 低8位IO口 和 高8位IO口

  • 从属于MODEy位用于控制对应编号的IO口方向

  • 从属于CNFy的用于控制对应编号的IO口工作模式
  1. 写输出内容（GPIO_ODR）
  • 配置GPIO_ODR的0-15位以分别配置16个IO口的输出电平状态

可读可写？
当IO口为输出模式时，GPIO_ODR的0-15位可以写数据，也可以读一下写了什么数据，但当IO口为输入模式时,只可读取，写入无效

  1. 范例程序——点亮PA5口LED
    int main(void){
    
    //配置时钟
    RCC_APB2ENR |= (0x01 << 2);

    //配置方向和模式(推挽输出 50MHz)

    //配置CNF5 = 00
    GPIO_CRL &= ~(0x0C << (5*4));

    //配置MODE5 = 11
    GPIO_CRL |= (0x03 << (5*4));

    //写输出内容（与置零 或置一）
    GPIOA_ODR &= ~(0x01 << 5);

    while(1){

    }
}
  • 注意点：
    1. 配置GPIO_CRL或GPIO_CRH寄存器时要注意它是四个位一组控制一个IO口，因此在按位移动的时候需要移动 IO口编号 * 4 个位数
    2. 用于置零的取反要最后进行
          //配置CNF5 = 00
    GPIO_CRL &= ~(0x0C << (5*4));

    /*
    右移：
        0000 1100 0000 0000…… 右移至目标位置
    取反：
        1111 0011 1111 1111…… 将CNF5置为00
    */

若改为：

    //配置CNF5 = 00
    GPIO_CRL &= (~0x0C << (5*4));

    /*
    取反：
        1111 0011
    右移：（右边补0！！！）
        1111 0011 0000 0000 ……
不仅CNF5被置为00，低位的MODE4 CNF4 …… MODE0 CNF0 也被置零
    */
    3. 与置零，或置一

通讯

通讯的类型

  • 串行/并行
    串行一条线：一次发一位
    并行多条线：一次发多位


  • 同步/异步
    同步：使用同一个时钟
    异步：分别使用各自时钟


  • 点对点/总线
    点对点：只能存在两部通讯设备，不需要设备地址
    总线：多设备，需要设备地址


  • 半双工/全双工
    半双工：线路允许双向传送资料，但同一时刻智能单向传输
    全双工：线路允许同时双向传送资料


  • 主从/对等
    主从：仅主机可以开启通讯，客机需要请求通讯（如：你和迅雷服务器）
    对等：众生平等，谁都可以开启通讯（如：BT传输）


  • 单端/差分
    单端：一个信号一条线
    差分：一个信号使用两条线传输（两线相位相同振幅相反），接收端比较两个电压的电压差来判断发送的逻辑状态，高度免疫外界电磁干扰

UART（通用异步收发器）

  • 书上的USART为 通用同步异步收发器

  • 使用UART通讯，除共地引脚之外，还需要至少两个引脚（RX TX）


  • 空闲状态、开始位和停止位

    1. 当空闲状态时，TX总为 高电平
    2. 开始位固定为 低电平
    3. 停止位固定为 高电平，且停止位占用位数可调

  • 数据的传输

    • 数据从最低位（LSB）开始发送，最后发送最高位（MSB）

  • 波特率（b/s）

    • 看单位就知道，指的是每秒传输的数据位

  • 校验位

    • 奇校验
      奇校验会把数据位中1的个数变为奇数，而不是判断是否为奇数
    • 偶校验
      奇校验会把数据位中1的个数变为偶数，而不是判断是否为偶数

  • 通讯协议

    • 通讯协议的写法为：
      波特率，数据位数 有无校验位 停止位占用位数
    • 如PPT写法：9600,8N1
      代表波特率9600，八位数据位，无校验位，停止位占用一位
有无校验位 写法 对应
无校验位 N null
奇校验 O odd
偶校验 E even

SPI（全双工同步串行通讯总线）

  • 使用SPI通讯，需要4个引脚

SS（CS）片选信号

  • 固定空闲状态为高电平，当变为低电平时即开始传输数据

SCLK时钟信号

  • 由主设备产生，有两个参数可调，对应四种模式

  • CPOL空闲电平对传输协议没有重大影响，只是决定空闲状态的电平，无论空闲状态是什么电平，开始传输的时候都是高低电平交替

SPI时钟模式 CPOL空闲电平 CPHA采样时刻
空闲低电平 第一个时钟边沿采样 0 0
空闲低电平 第二个时钟边沿采样 0 1
空闲高电平 第一个时钟边沿采样 1 0
空闲高电平 第二个时钟边沿采样 1 1

  • 当使用第一时钟边沿采样，当发送下一次数据时，SS必须先跳变至高电平再拉低，原因：

    • 可以从图中看到，在使用第一时钟边沿采样的情况下，SS被拉低时MOSI就已经加载第一个传输位 BIT7 并等待传输（等待对面的通讯机读取），第一个时钟传来意味着开始传输。


      若SS不跳变而时钟紧接着运作，则意味着没有第一个加载信号，MOSI将在第一个时钟的下降沿加载第一位，那第一个时钟的上升沿传输了个什么呢？可能是上一次传输的BIT0，这就有可能发生错误。


  • 当使用第二时钟边沿采样，当发送下一次数据时，SS可以不跳变，原因：
    • 可以从图中看到，在使用第二时钟边沿采样的情况下，SS被拉低时，并没有加载要传输的数据，而在时钟的第一边沿加载了第一个数据位 BIT7，第二个时钟边沿传输，这意味着加载和发送都由时钟控制，不受SS干涉，所以这时传输完毕就把时钟关掉，传输第二组数据就把时钟开启，完全不用管SS

UART通讯编程（Kinetis KL25）（基于寄存器）

  • 通讯总攻略：

    1. 开启端口时钟
    2. 开启UART模块时钟
    3. 设置IO口功能为UART通讯
    4. 配置UART相关寄存器

  • 这里使用的是PORTC3与PORTC4号端口，UART1

  1. 端口时钟配置（SIM）
// 开启C组端口时钟
SIM_SCGC5 |= (1 << 11);
  1. 配置UARTx模块时钟（SIM）
// 开启UART1模块时钟
SIM_SCGC |= (1 << 11);
  1. 设置IO口功能为UART通讯（PORTx_PCRn）
// 设置端口功能为UART通讯，选择3号功能即给MUX赋值011
PORTC_PCR3 &= ~(1 << 10);
PORTC_PCR3 |= (0x03 << 8);

PORTC_PCR4 &= ~(1 << 10);
PORTC_PCR4 |= (0x03 << 8);

  1. 配置UART相关寄存器（UARTx）

    1. 配置UART波特率（UARTx_BDL UARTx_BDH）

      • 总线时钟

        • PPT用单片机的总线时钟为：20.97 / 2MHz = 10.485MHz = 10485KHz = 10485e3Hz
        • 兆赫兹、千赫兹、赫兹的转换比为1000
        • 配置锁相环PLL可以超频总线时钟

      • 波特率的计算
        波特率寄存器里面存的数的计算公式为：

        波特率寄存器内数(10进制 取整) = 总线时钟(Hz) / (设置的波特率 * 16)

        如：
        配置为9600，则波特率寄存器内的数为：

        10485e3 / (9600 * 16) ≈ 68

        转为十六进制,即可应用至UARTx_BDL和UARTx_BDH：

        0x44
        • 写入寄存器的值再换算回波特率肯定不等于9600，但是波特率误差在5%以内是可以接受的

      • 配置UART波特率

        • BDL和BDH都要写入值，没有就写0，不能只写其中一个
          UART1_BDL = 0x44;
UART1_BDH = 0x00;

      • UART波特率计算函数

        // 定义使用的UART模块
#define UARTL UART1_BDL
#define UARTH UART1_BDH

// 定义总线时钟 设定的波特率 要写入的数
unsigned char busClock_khz = 10485;
unsigned char baudRate = 9600;
unsigned char sbr;

sbr = (unsigned short)((busClock_khz * 1000) / (baudRate / 16));

// 提取计算结果高八位 并 应用到UARTx_BDH
UARTH = (unsigned char)((sbr & 0x1F00) >> 8);
// 提取计算结果低八位 并 应用到UARTx_BDL
UARTL = (unsigned char)(sbr & 0x00FF);

    2. 配置UART控制寄存器（UARTx_C1 UARTx_C2）

      • 配置UART1_C1
      /*
主要是配置：
M：数据位位数（0：8位 1：9位）
PE：是否开启校验位（0：关闭 1：开启）
PT：校验位类型（0：奇校验 1：偶校验）
*/
UART1_C1 = 0;
      • 配置UART1_C2
      /*
主要配置：
TE：使能传输
RE：使能接收
*/
#define UART_C2_TE_MASK 0x08
#define UART_C2_RE_MASK 0x04

UART1_C2 |= (UART_C2_TE_MASK | UART_C2_RE_MASK);

    3. 配置UART状态寄存器——写入（传出）

      #define UART_S1_TDRE_MASK 0x80

void UART1_PutChar(unsigned char data) {
    // 当TDRE为0，循环检查等待
    while(!(UART1_S1 & UART_S1_TDRE_MASK));

    // 当TDRE为1，上一批数据传输完成，写入目前需要传输的数据
    UART1_D = data;
}

    4. 配置UART状态寄存器——读取

      #define UART_S1_RDRF_MASK 0x20

unsigned char UART1_GetChar(void) {
    // 当RDRF为0，循环检查等待
    while(!(UART1_S1 & UART_S1_RDRF_MASK));

    // 当RDRF为1，数据已经全部到达，读取该数据
    return UART1_D;
}

    5. 三个传输所需函数

      // 初始化
void uart1_init() {
    // 开启某组端口时钟
    SIM_SCGC5 |= (1 << 11);
    // 开启UART1时钟
    SIM_SCGC4 |= (1 << 11);

    // 配置端口功能为UART
    PORTC_PCR3 |= (3 << 8);
    PORTC_PCR4 |= (3 << 8);

    // 开启发送和接收

    UART_BDH |= 0x00;
    UART_BDL |= 0x44;
}

    // 当RDRF为0，循环检查等待
    while(!(UART1_S1 & UART_S1_RDRF_MASK));

    // 当RDRF为1，数据已经全部到达，读取该数据
    return UART1_D;
}

// 传出
void UART1_PutChar(unsigned char data) {
    // 当TDRE为0，循环检查等待
    while(!(UART1_S1 & UART_S1_TDRE_MASK));

    // 当TDRE为1，上一批数据传输完成，写入目前需要传输的数据
    UART1_D = data;
}

USART寄存器编程（STMF103CXXXX）

  1. 波特率设置（USART_BRR）

    1. 波特率计算

      • STM32波特率计算遵循以下公式：
        波特率寄存器内存的数（十进制浮点数） = 总线时钟（Hz） / ( 16 * 波特率（b/s） )

      如：已知fck总线时钟为72MHz，要得到115200波特率，USART_BRR寄存器内的值应该设置为多少？

      // 根据公式
波特率寄存器内存的数 = 72000000 / (16 * 115200)
波特率寄存器内存的数 = 39.0625

整数部分 十进制39 十六进制27
小数部分 十进制0.0625 十六进制0.0625 * 16 = 1

因此 寄存器内写入的数为 0x271
    • 波特率计算函数
      /*
* 方法名: setUsartBaudRate
* 作用: 设置USART_BRR波特率寄存器
* 形参: clk 传入总线时钟，单位为MHz
* 形参: baudRate 传入要设置的波特率
*/
void setUsartBaudRate(u32 clk, u32 baudRate) {
    // 定义要存在寄存器里的十进制数，整数部分，小数部分
    float hexNumber;
    u16 divMantissa;
    u16 divFraction;
    
    // 根据公式得到要存在寄存器里面的十进制数
    hexNumber = (float)(clk * 1000000) / (16 * baudRate);

    // 提取整数部分
    divMantissa = (u16)hexNumber;

    // 提取小数部分，并转换为十六进制格式
    divFraction = (hexNumber - divMantissa) * 16;

    // 将整数部分左移四位，给小数部分留位置
    divMantissa <<= 4;

    // 将值写入到寄存器
    USART1_BBR = divMantissa + divFraction;
}
    1. 波特率的写入（USART_BRR）
      • STMF103需要配置USART_BRR波特率寄存器来设定波特率，其中，该寄存器的15~4位存放波特率整数，3~0位存放波特率小数

  1. 配置USART控制寄存器（USART_CR1）

    • 主要需要配置：
      1. EE：使能USART（0：禁用 1：启用）
      2. M：数据位长度（0：8个数据位 1:9个数据位）
      3. PCE：校验位开关（0：禁用 1：启用）
      4. PS：校验位类型（0：偶校验 1： 奇校验）
      5. TE：发送使能（0：禁用 1：启用）
      6. RE：接收使能（0：禁用 1：启用）
        // 配置为：USART启用 8数据位 禁用校验位 发送使能 接收使能
USART1_BRR |= 0x200C;

中断（Kinetis KL25）

中断与轮询

中断：又硬件判断是否发生外部事件并在发生时同时CPU

轮询：周期性地/连续性地检查外部事件是否发生

中断向量表

  • 中断向量表是一段连续的存储空间
  • 在复位后有磨人的起始位置
  • 每个中断在向量表中都有相应的表项，该表值为该中断对应的服务程序的地址
  • 有特定的程序代码确定中断向量表的每个表项

  • 中断总攻略：

    2. 配置中断源
    3. 编写中断服务子程序

    1. 打开总中断

  1. 打开中断

    1. 总中断使能/失能（NVIC_ISER[0/1]/NVIC_ICER[0/1]）
    • STM32F103CXXXX共有60个可屏蔽中断，因此用到了两个32位寄存器来控制使能，两个32位寄存器来控制失能
      // 使能7号中断

    1. 优先级分组（SCB_AIRCR）

    • 这个分组旨在将寄存器NVIC_ICRx的高四位分为抢占优先级x位，子优先级y位

    • 由SCB_AIRCR寄存器的8~10位控制

    SCB_AIRCR(10:8) 红线位置 抢占优先级位数 子优先级位数
    111 7之后 无 7~4
    110 7/6之间 7 6~4
    101 6/5之间 7~6 5~4
    100 5/4之间 7~5 4
    011 4/3之间 7~4 无
    • 并不是直接分级，并不是直接分级，并不是直接分级

    分组决定了分级可以取的值，例如：红线在6/5之间即抢占优先级和子优先级各占两位，这意味着抢占优先级和子优先级都可以取0~3

    // 写入优先级分组函数
void NVIC_PriorityGroupConfig(u8 NVIC_Group) {
    u32 groupConfig, writeIn;
    // 取NVIC_Group后三位并将其移到10~8位
    groupConfig = (~NVIC_Group) & 0x07;
    groupConfig <<= 8;
    // 读取寄存器原设置（为了在写入完整钥匙的同时其他设置不改变）
    writeIn = SCB_AIRCR;
    /* 清空PRIGROUP原设置和秘钥
    因为读取的秘钥与写入的秘钥不同，见课本P54 */
    writeIn &= 0x0000F8FF;
    // 写入秘钥
    writeIn |= 0x05FA0000;
    // 写入组设置
    writeIn |= groupConfig;
    // 将全部设置应用到寄存器
    SCB_AIRCR = writeIn;
}

    1. 抢占优先级和响应优先级（数字越小优先级越高）
    • 抢占优先级：打断其他中断的权利，抢占优先级高的可以打断抢占优先级低的中断，同级不可互相打断
    • 响应优先级：若两个中断同时到达，CPU优先响应的权利（但不影响它会被高抢占优先级的中断打断）
    // NVIC初始化函数
void NVICInit(u8 NVIC_PreemptionPriority, u8 NVIC_SubPriority, u8 NVIC_Channel, u8 NVIC_Group) {
    u8 PriorityConfig;
    // 设置分组
    NVIC_PriorityGroupConfig(NVIC_Group);
    // 写入抢占优先级
    PriorityConfig = NVIC_PreemptionPriority << (4 - NVIC_Group);
    // 写入响应优先级
    PriorityConfig |= NVIC_SubPriority & (0x0F >> NVIC_Group);
    // 取低4位
    PriorityConfig &= 0x0F;
    // 使能中断位
    NVIC_ISER[NVIC_Channel / 32] |= (1 << (NVIC_Channel % 32));
    // 设置响应优先级和抢断优先级
    NVIC_IP[NVIC_Channel] |= PriorityConfig << 4;
}

  2. 配置对应中断触发方式

    // 配置中断触发方式与映射
void Ex_NVIC_Config(u8 GPIOx,u8 BITx,u8 TRIM) 
{
    u8 EXTADDR;
    u8 EXTOFFSET;
    EXTADDR=BITx/4;//得到中断寄存器组的编号
    EXTOFFSET=(BITx%4)*4;

    EXTI->IMR|=1<<BITx;//  开启line BITx上的中断
    //EXTI->EMR|=1<<BITx;//不屏蔽line BITx上的事件 (如果不屏蔽这句,在硬件上是可以的,但是在软件仿真的时候无法进入中断!)
    if(TRIM&0x01)EXTI->FTSR|=1<<BITx;//line BITx上事件下降沿触发
    if(TRIM&0x02)EXTI->RTSR|=1<<BITx;//line BITx上事件上升降沿触发

SysTick系统定时器

  • SysTick是一个 ARM自制的 24位 递减 的定时器，它在每个cortex系列都有
  • 它由 4个 寄存器控制，由于校准值寄存器（STK_CALRB）不常用，因此不做说明

控制寄存器STST_CSR

  • 该寄存器有4位有意义：
    bit 0：ENABLE SysTick使能位
    bit 1：TICKINT：SysTick中断使能位
    bit 2：CLKSOURCE：时钟源选择（0：使用HCLK/8作为时钟源 1：使用HCLK作为时钟源）（HCLK见P30 最大72MHz）
    bit 16：COUNTFLAG，若在上一次读取本寄存器后，定时器再次倒数至0，则该位自动置1，读取该位会使其自动清零

重载寄存器STST_RVR

  • 当定时器递减至0时，重载寄存器中的值就会被重装载，继续开始新一轮递减
  • 重载寄存器为24位寄存器，最大值为0xFFFFFF

当前值寄存器STST_CVR

  • 读取时返回当前技术的值，写该寄存器则会将寄存器清零，同时清空COUNTFLAG标志位
  • 24位寄存器

中断时间计算

公式为：(重载值 + 1) * (1 / 时钟源频率)

编程实例

// 开启全局中断
asm(*CPSIE i*);

// 写入重装载值

// 清除当前值寄存器

// 使能SysTick与中断，选择时钟源
STST_CSR = 0x07;

    uint8_t i;
    // write code here

    // 读当前值寄存器以清除标志位

}

TIM通用定时器

  • 开定时器步骤：
    1. 使能TIM时钟
    2. 设置分频
    3. 设置重装载
    4. 使能中断
    5. 使能计数器

计数器使能TIM_CR1

  • 将第0位置1即可使能计数器

中断使能TIM_DIER

  • 将第0位置1即可使能中断

分频设置TIM_PSC

分频的计算公式：计数器+1频率 = 时钟 / (预分频器数值 + 1)


重装载设置TIM_ARR

重装载的计算公式：定时时间 = (重装载值 + 1) * (1 / 计数器+1频率)


中断标志位TIM_SR

  • 在中断子程序最后要清除中断标志位

ADC

模数转换的过程（重要）

  • 模数转换分三个过程：
    1. 采样: 根据设定频率在特定的时间对被采样信号进行采样

      采样定律：采样频率必须大于等于被采样信号频率的两倍方可复原信号

    2. 量化: 采样锁得幅值必须转化为某个规定的最小单位的整数倍
    3. 编码: 用n位的二进制码表示量化后的结果，这就是ADC最后的输出结果

模数转换性能参数

  • 量程：允许输入的电压范围
  • 分辨率：ADC能分辨的最小电压变化量
  • 精度：ADC输出的实际值与理论值之差

ADC转换时间

所需转换时间 = 采样时间 + 12.5个周期(量化+编码时间)

一道例题

  • 一个12位的ADC，输入电压范围是0~5V，假设输入模拟量和输出数字量是线性关系，先输入模拟量2V，输出数字量为多少？

    (2/5) * 4095 = 1638

关于ARM Cortex的32位MCU

  • 32位总线意味着可以 单次读写 4字节 的数据

    32位总线，可以将它看成一条有32条线的并行传输线（应该就是）
    1条线传1个位，32条线32位，1字节 = 8位
    所以是 4字节

  • 32位，总共可访问地址为 4G

    可访问地址范围从0x0000 0000到0xFFFF FFFF
    也就是2^32种可能，每一个地址对应一个B(byte)

    1024 = 2^10
    1024B = 1KB, 1024KB = 1MB, 1024MB = 1GB

    换算过来就是2^32(B) / 2^10(转化为KB) / 2^10(转化为MB) / 2^10(转化为GB) = 2^2 = 4 GB

程序计数器PC

PC永远指向下一条要执行的语句，若遇到跳转到子程序，则指向子程序地址

程序状态寄存器

单片机执行的计算若有溢出，借位等都会将程序状态寄存器中的对应位置1
