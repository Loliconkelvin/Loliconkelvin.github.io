第一个JDBC程序

一个JDBC程序分为以下几个部分：

  1. 加载驱动
  2. 获取连接，获取数据库对象
  3. 获取执行SQL的对象
  4. 执行SQL
  5. 返回结果集
  6. 关闭连接
public static void main(String[] args) {
    Connection connection = null;
    Statement statement = null;
    ResultSet resultSet = null;

    String url = "jdbc:mysql://localhost:3306/test_db_for_jdbc?" +
            "&useUnicode=true" +
            "&characterEncoding=utf8" +
            "&useSSL=true" +
            "&serverTimezone=UTC";
    String user = "root";
    String password = "123456";

    String sql = "SELECT * FROM users";

    String sql2 = "SELECT * FROM users WHERE id = 1";

    try {
        // 加载驱动
        Class.forName("com.mysql.cj.jdbc.Driver");

        // 连接，返回数据库对象
        connection = DriverManager.getConnection(url, user, password);

        // 获取执行SQL的对象
        statement = connection.createStatement();

        // 执行SQL
        resultSet = statement.executeQuery(sql);

        while (resultSet.next()) {
            System.out.println(
                    resultSet.getObject("id").toString()
                            + resultSet.getObject("name")
                            + resultSet.getObject("password")
                            + resultSet.getObject("email")
                            + resultSet.getObject("birthday"));
        }


    } catch (ClassNotFoundException | SQLException e) {
        e.printStackTrace();
    } finally {
        // 关闭连接
        try {
            if (resultSet != null) {
                resultSet.close();
            }
            if (statement != null) {
                statement.close();
            }
            if (connection != null) {
                connection.close();
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

JDBC常用对象详解

DriverManager

作用1-加载驱动：加载驱动有两种方式：

// 1. 使用方法初始化
DriverManager.registerDriver(new com.mysql.cj.jdbc.Driver());

// 2. 直接加载类
Class。forName("com.mysql.cj.jdbc.Driver");

作用2-获取数据库对象：

使用 DriverManager的对象.getConnection() 方法获取数据库对象，该方法有两种可传入参数：

  • getConnection(String url)：在url处指定用户名和密码

  • getConnection(String url, String user, String password)：不在url处指定用户名和密码

URL

用于指明数据库所在的地址和各种连接参数

使用以下格式定义URL以被用于连接数据库：

// 连接MySQL（端口：3306）
String url = "jdbc:mysql://地址:3306/数据库名?参数1&参数2&...";

// 连接Orecle（端口：1521）
String url = "jdbc:orecle:thin@localhost:1521:sid";

关于参数：

  • user=用户名：在url指定用户名

  • password=密码：在url指定密码

  • useUnicode=true：可使用中文

  • characterEncoding=utf8：这也要解释？

  • useSSL=false：是否建立安全的连接（详见 mysql数据库的useSSL=false或者useSLL=true参数设置的区别）

  • serverTimezone=UTC：服务器时区设置

Connection

这个类的对象代表一个数据库，它由 DriverManager 对象的 getConnection() 方法获得，因此他可以设置数据库的一些参数和操作整个数据库，如：setAutoCommit()、commit、rollback()

Statement

这个类的对象代表一个用于在该数据库执行SQL语句的对象，他可以传入SQL语句

使用 Statement 类的对象调用以下方法实现SQL语句的执行：

  • execute(String sql)：什么SQL语句都可以执行

  • executeQuery(String sql)：执行查询，返回结果集，结果集使用 ResultSet 接收

  • executeUpdate(String sql)：执行执行插入、更新和删除，返回受影响的行数

当然你可以使用 getResultSet() 方法获取结果集

ResultSet

是一个用于接收查询SQL运行完成返回的结果集的类，他封装了所有查询结果

使用 ResultSet 类的对象调用以下方法实现对结果集内指定字段的值的提取：

  • getObject(String columnLabel)：获取结果集中当前行字段名为columnLabel的值，其类型未知

  • getInt(String columnLabel)：获取结果集中当前行字段名为columnLabel的值，其类型为 INT

  • getFloat(String columnLabel)：获取结果集中当前行字段名为columnLabel的值，其类型为 FLOAT

  • …

结果集的遍历：

还记得 List 吗，Iterator呢，对，就是那个迭代器，ResultSet自身就可以看成一个迭代器，它可以使用以下方法移动它的指针：

beforeFirst()：移动到最前面
afterLast()：移动到最后面
next()：移动到下一行数据
previous()：移动到上一行数据
absolute(row)：移动到指定行

关闭资源

需要关闭三个连接，分别是：

  1. 代表结果集的对象 ResultSet 类的对象

  2. 执行SQL语句的对象 statement 类的对象

  3. 代表数据库的对象 Connection 类的对象

提取工具类

因为在Java与数据库交互时只有SQL语句和结果在变，因此若每次执行语句都写一次连接数据库的代码，代码量就会很大，因此我们需要创造一个工具类

配置文件

若把连接数据库的信息直接写在代码里面，则连接数据库功能的耦合性就会很高，因此配置需要放在properties文件里面（properties类属于 Java集合框架->Properties），然后通过反射的方式读取文件，下面细说。

配置文件的写法：

配置文件为 Properties 文件，其文件内容格式为：

注意不包含任何引号和分号

键名1=值1
键名2=值2
...

获取配置文件：

使用反射获取类加载器的方法获取配置文件：

// 返回一个输入流
当前类.class.getClassLoader().
getResourceAsStream("完整资源名");

注意：

若不是Maven项目，则类加载器加载的资源为 当前项目src目录下的文件，因此：

  • 若资源文件在src目录下，则资源名处直接写 文件名
  • 若资源文件在src目录的子目录下（也就是src下还有一个文件夹），则资源名处写 文件夹名/文件名
  • 若资源在src目录外，则获取不到

若是Maven项目，需要将资源文件丢到项目自带的 resources 目录下

读取配置文件：

使用 Properties 类的对象中 load(InputStream in) 方法读取该输入流

Properties properties = new Properties();
properties.load(in);

使用 Properties 类的对象中 getProperty(String key) 方法读取配置文件中的内容

driver = properties.getProperty("driver");
url = properties.getProperty("url");
user = properties.getProperty("user");
password = properties.getProperty("password");

工具类中的静态代码块

工具类的静态代码块要做的有两件事：

  1. 读取配置文件
  2. 加载驱动
static {
    try {
        // 加载配置文件
        InputStream in = JdbcUtil.class.getClassLoader().getResourceAsStream("db.properties");

        Properties properties = new Properties();
        properties.load(in);

        driver = properties.getProperty("driver");
        url = properties.getProperty("url");
        user = properties.getProperty("user");
        password = properties.getProperty("password");

        // 驱动只用加载一次
        Class.forName(driver);
    } catch (IOException | ClassNotFoundException e) {
        e.printStackTrace();
    }
}

释放连接方法

// 释放连接
public static void close(Connection conn, Statement st, ResultSet rs) {
    if (rs != null) {
        try {
            rs.close();
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
    if (st != null) {
        try {
            st.close();
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
    if (conn != null) {
        try {
            conn.close();
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

SQL注入问题

上面这么多程序都只是SQL语句在变，结果在变，那我们可以不可以让用户传我们指定的字段的值，然后使用String拼接成一个SQL语句呢？如下：

// SELECT * FROM users WHERE name = ' ' AND password = ' '
//                                   ^                  ^
String sql = "SELECT * FROM users WHERE name = '" + name + "'"
        + "AND password = '" + pwd + "'";

可以是可以，但如果我输入的账号和密码分别是 zs' OR '1=1 和 321' OR '1=1 呢？那么SQL语句就会变成：

SELECT * FROM users WHERE name = 'zs' OR '1=1'
AND password = '321' OR '1=1';

因为1=1永远为真，所以所有数据都会被筛选出来


还有更牛逼的，输入账号为 zs' OR TRUE #，密码随意，则SQL语句会变成：

SELECT * FROM users WHERE 'TRUE' OR name = 'zs' OR TRUE #' AND password = '321';

后面直接被注释，同样是可以筛选所有数据

为了解决这个问题，我们将使用 PreparedStatement 类

PreparedStatement类

PreparedStatement类相比于Statement类，可以有效防止SQL注入，并且效率更高

PreparedStatement对象

使用 Connection 类的对象中 prepareStatement(sql) 方法创建对象。

怎么在这里就传SQL语句？因为SQL需要在这里先预编译（此时不执行），如果使用过commons-dbutils，那这个预编译需要写的SQL语句一定倍感亲切！

预编译SQL语句和传参

SQL语句：

预编译SQL语句的写法就是在需要传入参数的地方使用占位符 ? 代替，如：

UPDATE users SET id = 25 WHERE name = ?

参数传入：

使用 PreparedStatement 类的对象中 setXXX(int parameterIndex, XXX x) （XXX代表数据类型）来给预编译的SQL语句赋值。

  • parameterIndex 代表参数下标，参数下标从1开始

执行语句

使用 PreparedStatement 类的对象中以下方法执行SQL语句

executeQuery()：执行查询

executeUpdate()：执行删除、更改、插入

JDBC与事务

事务的概念（ACID原则）

（A）原子性：要么全部完成，要么全部不完成
（C）一致性：数据库处理前后的结果应该与其所抽象的客观世界中的真实情况保持一致（电脑处理的结果要和现实的结果一样
（I）隔离性：多个进程互不干扰
（D）持久性：一旦提交不可回滚，持久化到数据库中

其中隔离性会导致以下几个问题：

  • 脏读：一个事物读取了另一个没有提交的事务
  • 不可重复读：在同一个事务内，重复读取表中的数据，表数据发生了改变
  • 虚读（幻读）：在一个事务内，读取到了别人插入的数据，导致前后读出的结果不一致

在IDEA实现事务的提交与回滚

开启事务：

使用 Connection 类的对象中 setAutoCommit(false) 方法关闭自动提交，关闭自动提交即为开启事务

conn.setAutoCommit(false);

提交与回滚：

使用 Connection 类的对象中 commit() 方法进行事务提交

使用 Connection 类的对象中 rollback() 方法进行事务回滚

  • 注意：你可以在异常捕获处显式定义事务回滚，但默认的设置是，在事务提交之前，若遇到任何错误，都会自动回滚
conn.commit();

conn.rollback();

数据库连接池

数据库连接池概念

还记得线程池（Java-JUC->线程池）吗，连接池与线程池的作用大体相同，都是利用池化技术，提前创建好需要使用的资源，当需要使用时，直接用池里面的资源。

与线程池类似，数据库连接池一般包含如下设置：

  • 最小连接数（对应线程池的核心线程池大小）：表示初始就会有n条连接被创建，等待被使用
  • 最大连接数（对应线程池的最大线程池大小）：当初始denn条连接都被使用时，还有连接接入，则额外创建连接，但连接的数量不会超过最大连接数
    若连接达到了最大连接数，还有连接接入，则排队等待
  • 等待超时：指定排队等待的线程的等待时间

目前常用的连接池有 DBCP2 C3P0

DBPC2连接池

前期准备：

使用DBCP2连接池，需要使用以下jar包

commons-dbcp2-版本号.jar
commons-pool2-版本号.jar
commons-logging-版本号.jar（注意DBCP2额外需要用到这个包，DBCP第一代不需要）

将他们复制到与src同级别的lib目录下（没有则创建）并 Add as library

配置文件：

dbcp2使用读取properties的输入流的方式读取配置文件

  1. 在src目录下创建一个properties文件，下面是一个properties文件内容的例子：

    # 连接设置
driverClassName=com.mysql.cj.jdbc.Driver
url=jdbc:mysql://localhost:3306/test_db_for_jdbc?useUnicode=true&characterEncoding=utf8&useSSL=false
username=root
password=123456

# 最小连接数
initialSize=10

# 最大连接数
maxActive=50

# 最大空闲连接（当连接数被扩张并且使用完毕时，先不释放多余的空闲连接）
maxIdle=20

# 最小空闲连接
minIdle=5

# 超时等待时间/毫秒（超过此时间才释放多余的空闲连接）
maxWait=6000

# JDBC驱动连接时附带的连接属性[格式：属性名=参数;]
connectionProperties=useUnicode=true;characterEncoding=utf8

# 指定由连接池所创建的连接的自动提交状态
defaultAutoCommit=true

# 指定由连接池所创建的连接的只读状态
# 若没有设置该值，则‘setReadOnly’方法不会被调用
defaultReadOnly=

# 指定由连接池所创建的连接的事务级别
defaultTransactionIsolation=READ_UNCOMMITTED

  2. 同样在静态代码块中使用类加载器的方法获取配置文件

  3. 使用 BasicDataSourceFactory 类中的静态方法 createDataSource(Properties properties) 创建 dataSource 对象

static {
    try {
        InputStream in = JdbcUtil_DBCP.class.getClassLoader().getResourceAsStream("dbcpconfig.properties");
        Properties properties = new Properties();
        properties.load(in);

        dataSource = BasicDataSourceFactory.createDataSource(properties);
    } catch (IOException e) {
        e.printStackTrace();
    } catch (Exception e) {
        e.printStackTrace();
    }
}
  1. 使用 dataSource 对象中的 getConnection() 方法获取连接
public static Connection getConnection() throws SQLException {
    return dataSource.getConnection();
}

  1. 获取到连接对象之后就和上面玩法一样了，创建 PreparedStatement 对象，执行SQL之后返回 ResultSet 然后遍历结果。

  2. 使用 dataSource 对象中的 close() 方法断开连接

public static void close() throws SQLException {
    dataSource.close();
}
