注释

行注释

  1. 说明：
    行注释对一行进行注释，适合注释字数少的时候使用
  2. 快速输入：
    无
  3. 代码规范：
    • 方法内部单行注释，在被注释语句上方另起一行，使用//注释。方法内部 多行注释使用/* */注释，注意与代码对齐
    • 注释后面跟一个空格
      // 这里写行注释

段落注释

  1. 说明：
    段落注释允许写多行注释，适合注释字数多的时候使
  2. 快速输入：
    /* + enter
  3. 代码规范：
    无
    /*
这里写段落注释
*/

  1. 说明：
    详见：https://blog.csdn.net/vbirdbest/article/details/80296136
  2. 快速输入：
    /** + enter

    • 类、类属性、类方法的注释必须要使用Javadoc规范，禁止使用行注释
    • 所有的抽象方法（包括接口中的方法）必须要用 Javadoc 注释,除了返回值、参数、异常说明外，还必须指出该方法做什么事情，实现什么功能。

标识符和关键字

标识符（含变量、方法、类命名风格）

  1. 说明：
    Java的所有组成部分都需要名字。类名，变量名和方法名都被称为标识符
  2. 代码规范：
    • 代码中标识符的命名均不能以下划线或美元符号开始，也不能以下划线和美元符号结束。
    • 禁止拼音英文混合命名和纯中文命名
    • 类名使用单词首字母大写形式。（如：XmlService）
    • 方法名、参数名、成员变量、局部变量使用驼峰命名。
    • 常量命名字母全部大写，单词间用下划线隔开。

      关键字

  3. 说明：
    Java关键字是电脑语言里事先定义的，有特别意义的标识符，有时又叫保留字，还有特别意义的变量。 Java的关键字对Java的编译器有特殊的意义，他们用来表示一种数据类型，或者表示程序的结构等，关键字不能用作变量名、方法名、类名、包名和参数。

获取输入（Scanner）

  1. 新建Scanner对象以接收输入的数据

    • 直接通过补全输入以下语句后会自动包含java.util.Scanner工具包
      新建了一个对象用于接收用户的输入，若有用户输入，则将输入装载进scan对象里
      // scan对象会接收用户的输入内容
Scanner scan = new Scanner(System.in);

  2. 关闭Scanner（重要）

  • 每次使用完Scanner后请关闭Scanner

  • 可以在新建好Scanner的对象之后先写上关闭，然后在新建和关闭中间写操作

    scan.close();

  1. 判断是否有输入（非必须）

  • hasNext()

    if(scan.hasNext()){
    // do something
}
    • hasNext()方法会检测scan对象是否为非空字符，但并不会将scan里面的内容提取出来
      若scan里面不是非空字符，则返回true
      若scan里面是非空字符，则阻塞方法，等待用户输入
    • 注意：
      使用检测之后一定要用提取将scan对象里的内容拿出来，否则hasNext会一直返回true

  • hasNextLine()

    if(scan.hasNextLine()){
    // do something
}
    • hasNext()方法会检测scan对象是否有一行（包括空行），但并不会将scan里面的内容提取出来
  1. 提取（使用）输入的数据

  • 使用 Scan（Scanner对象）.next() 即可提取里面的内容，里面的值可以直接使用也可以赋给一个变量

    • 注意：next在遇到空格（包括Tab）的时候会视为结束符，并且会把有效字符前面的空格过滤掉
      （或许可以用来接收一个字符之类的，只需要规定字符后面有空格，如接收2 + 3的“ + ”）
    package com.company.demo;

import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner scan = new Scanner(System.in);
        System.out.println("你输入的是：" + scan.next());
        scan.close();
    }
}

// 输入：   hahah hhhhhh（前面有空格）
// 输出：你输入的是hahah

// 输入：hahahah hhhhhhh
// 输出：你输入的是hahahah

  • 使用 Scan（Scanner对象）.nextLine() 即可提取Scan中的一整行内容

    • 注意：nextLine的结束符只能是Enter，因此他会把回车前面的内容全部提取，它可以接收空格
    package com.company.demo;

import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner scan = new Scanner(System.in);
        System.out.println("你输入的是：" + scan.nextLine());
        scan.close();
    }
}

// 输入：   hahah hhhhhh（前面有空格）
// 输出：你输入的是   hahah hhhhhh

带有判断性的检测/只接收指定格式的提取

  • 整型：

    package com.company.demo;

import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner scan = new Scanner(System.in);

        int num01;
        float num02;

        System.out.println("请输入整数：");
        if (scan.hasNextInt()) {
            num01 = scan.nextInt();
            System.out.println("你输入的是：" + num01);
        }

        scan.close();
    }
}

  • 同样支持hasNextxxx与nextxxx的有：
    Float byte short double等，但String没有，因为它不是基本数据类型
    其中double和float可以兼容检测int

  • hasNextxxx的判断：hasNextxxx与hasNext不同，它检测到了不同类型的值不会等待用户输入一个正确的值，而是返回false

  • 一个接收数据并求平均数的程序：

    package com.company.demo;

import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner scan = new Scanner(System.in);

        int i;
        double total = 0;
        double result;

        for (i = 0; scan.hasNextDouble(); i++) {
            total += scan.nextDouble();
        }

        if (i == 0){
            System.out.println("未输入数字");
        }

        result = total / i;

        System.out.println("result = " + result);

        scan.close();
    }
}

避免死循环

当遇到scan（Scanner对象）造成的死循环时，多数是因为scan里面的数据没有被提取出来，只需要使用scan.next();提取但不使用里面的内容即可。

数组的特点

  1. 长度确定，一旦分配了内存就不可以改变长度
  2. 其成员必须是相同类型（物以群分，这就是数组啊）
  3. 数组的类型可以是任何类型，包括基本类型和引用类型（自己创建的类咯）
  4. 数组本身就是对象，Java的对象是在堆中的（因为是new出来的），因此无论是原始类型还是引用类型，数组对象是保存在堆中的
    • 这里关于数组对象保存在堆中，下面又说numbers是变量需要进栈怎么理解呢：
      只能是说numbers是一个变量，其指向的是一个在堆中占用了内存的数组对象，因此下面地址=0x0011那个框才是数组对象

    动态初始化数组就是先分配内存（指定长度）（并且自动给每个成员赋默认值），然后再给每个成员赋值
    静态初始化数组就是一开始就往数组里面塞成员，塞多少个，长度就是多少
  1. 声明 -> 分配空间 -> 赋值（动态初始化）

// 赋值

  1. 声明并分配空间 -> 赋值（动态初始化）（常用）
  • 没有赋值的元素默认为0或者null
// 声明并分配空间 new int[10].var

关于数组创建的内存分析

堆与栈

  • 栈
    存储局部变量（如定义在方法内的变量、for循环的变量），更新速度快（局部变量生命周期短，一旦变量在栈的生命周期结束即马上被释放）
  • 堆
    存放对象（如String类型、用到new创建的任何对象），当一个实体没有被 引用数据类型（详见分配空间 指向的时候即视为垃圾，会不定时被Java垃圾回收机制回收

数组在内存中被创建的过程

  • 当我们执行下面一串代码时： 
    public class Main {
    public static void main(String[] args) {
        int[] number;
        number = new int[10];
        number = null;
    }
}

  1. 进栈
    当执行 int[] number; 时，即新增了一个局部变量，因此需要将局部变量进栈


    要存储局部变量，要先将包含该局部变量的方法线进栈,然后将局部变量进栈。

  1. 分配空间
    当执行 number = new int[10]; 时，即确定了数组的长度，该语句为new语句，意思是需要在堆中给new出来的对象分配一块内存。而局部变量number则是指向这块内存的一个引用数据类型（是你！指针！）

  • 所以 int[] number = new number[10]; 只是将这两句话缩写一下罢了
  1. 取消指向
    当执行 number = null; 时，引用数据类型number被置空，指向虚无，则原来被number指向的堆中地址为0x0011的空间已经没有变量指向它，因此它无法被引用，无法再被使用，也就成为了垃圾，会被Java垃圾回收机制回收。

数组常见异常

ArrayIndexOutOfBoundsException:数组下标越界
意思是某处出租下标搞错了，如一个数组num有9个成员，代码确有一句 num[9] = 10; （因为数组从0开始数，因此9个元素应该是0-8，不存在下标为9的元素）
