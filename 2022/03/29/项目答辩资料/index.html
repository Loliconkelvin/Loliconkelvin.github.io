前端篇

  • 登录界面的制作理念？

    登录界面、甚至是整个网页界面，我们都希望是简洁圆润的，可以让人一眼就看到想看见的东西，马上就能明白什么按钮有什么用

  • 悬浮窗是怎么实现的？

    悬浮窗它首先是一个框，所以我们画一个白底圆角矩形

    其次他一开始是隐藏的，所以我们display: none

    然后他还会跟着滚动条的一定而移动，因此是绝对定位/固定定位（fixed）

    关于fixed和absolute：在有滚动条的情况下，fix不会随滚动条移动而移动，absolute会。absolute会随着窗口大小改变而改变

    最后是动画效果，通过在一个按钮上绑定一个click事件，使用jquary里面的slideDown/Up方法实现悬浮窗像一个卷轴一样滑下来

  • 在页面取到个人信息是怎么实现的？

    一个网页应该尽可能的避免出现过多的后端代码，比如在jsp里面不应该出现java代码，通过EL表达式实现java代码的简化，

    从我们打开这个服务器开始，就会产生一些盒子（容器）来装数据：

      1. Application（存在于服务端 服务器启动->关闭）
  2. Session（存在于双端 用户访问该站点（服务器）-> 用户离开）
  3. Request（存在于客户端 每次请求都会刷新）

    EL表达式会自动从这三个区域寻找数据

  • jquery是什么

    （上面说到EL表达式是简化页面里的java代码的，那么）jquery是用来简化js代码的。

    js代码如何获得一个对象？

    document.getElementById();

    jquery:$("")

后端篇

Tomcat

  • 什么是Tomcat？

    Tomcat是一个免费轻量级的web服务器，普遍用在一些中小型系统的架设，可以说它是jsp网页和后端servlet的容器

  • 什么是Servlet？

    Servlet一个运行在服务器上并且用于 与前端交流，使web页面动态化 的小程序

项目相关

  • 业务的实现思路？（同步）

    业务主要分为前端和后端

    前端的活：

      1. 收集（表单）数据
  2. 指向一个servlet业务
  3. 给该servlet业务传入参数
  4. 收到后端回应后根据回应做出相应动作（如提示，跳转）

    后端的活：

      1. servlet层收集前端传入的参数
  2. 将参数传入业务层
  3. 业务层传入数据库交互层
  4. 数据库交互层根据参数进行数据库查询并返回结果打哦业务层
  5. 处理结果（如执行一些判断）并返回给servlet层
  6. servlet层根据结果利用resp回应前端

  • 怎么实现后端和数据库交互？

    通过 数据库连接池c3p0和dbutils这些jar包（类库） 实现后端和数据库的交互

    c3p0是一个用于自动管理数据库连接的jar包，我们可以把一个个连接看成一条条线，c3p0将这些线放在一个盒子中管理，已经不通讯的链接（线）会被自动回收

    dbutils是一个专门用于数据库连接、操作（包括接收器等）的jar包

    通过dbutils里QueryRunner类的对象来执行数据库查询
