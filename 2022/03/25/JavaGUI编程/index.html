JavaGUI编程

一、JavaAWT

下面是AWT涉及类的继承关系

组件和容器

窗口：窗口是一个顶层容器，可以在窗口里面直接添加组件，也可以添加面板

  • 使用下面的代码画一个窗口 
    public class TestFrame {
    public static void main(String[] args) {
        // Frame
        Frame frame = new Frame("Java图形界面");

        // 设置可见性
        frame.setVisible(true);

        // 设置窗口大小
        frame.setSize(400,400);

        // 设置背景颜色
        frame.setBackground(new Color(51, 218, 66));

        // 弹出初始位置
        frame.setLocation(200,200);

        // 设置不可变形
        frame.setResizable(false);
    }
}

面板：面板是一个中间容器，它可以被添加在窗口内的同时装下一些组件

注意：
1. 窗口的位置设定是相对于屏幕而言的
2. 面板的位置设定是相对于窗口而言的

  • 使用以下代码实现一个面板（Panel）并将它放在窗口内
public class TestPanel {
    public static void main(String[] args) {
        Frame frame = new Frame();

        // 布局问题
        Panel panel = new Panel();
        // 设置布局
        frame.setLayout(null);
        // 设置位置和大小
        frame.setBounds(300, 300, 500, 500);
        frame.setBackground(new Color(255, 0, 0));

        // 相对于frame的位置
        panel.setBounds(50, 50, 400, 400);
        panel.setBackground(new Color(44, 32, 32));

        frame.add(panel);

        frame.setVisible(true);
    }
}

布局管理器

流式布局：流式布局指将组件以最佳尺寸一个个按照加入顺序 从左往右 自上而下 排列

改变窗体大小并不会改变组件大小，但是可能会改变组件的位置

  • 使用以下代码实现一个流式布局窗口 
    public class TestLayout {
    public static void main(String[] args) {
        Frame frame = new Frame();

        Button button1 = new Button("button1");
        Button button2 = new Button("button2");
        Button button3 = new Button("button3");

        // 设置流式布局
        frame.setLayout(new FlowLayout());
        frame.setSize(200,200);
        frame.setVisible(true);

        frame.add(button1);
        frame.add(button2);
        frame.add(button3);
    }
}

流布局除了无参构造之外，还可以通过加入参数来指定流布局的对齐方式，如：

  • new FlowLayout(FlowLayout.RIGHT);

边框布局：边框布局指将该容器分为东西南北中五块区域

  • 通过以下代码实现一个边框布局的窗口： 
    public class TestLayout {
    public static void main(String[] args) {
        Frame frame = new Frame();

        Button center = new Button("center");
        Button north = new Button("north");
        Button south = new Button("south");
        Button west = new Button("west");
        Button east = new Button("east");

        // 设置边框布局
        frame.setLayout(new BorderLayout());

        frame.add(center, BorderLayout.CENTER);
        frame.add(north, BorderLayout.NORTH);
        frame.add(south, BorderLayout.SOUTH);
        frame.add(west, BorderLayout.WEST);
        frame.add(east, BorderLayout.EAST);

        frame.setSize(200, 200);
        frame.setVisible(true);
    }
}

表格布局：表格布局指件窗口分隔为指定的行列数，表格内组件会根据添加顺序 从左往右 自上而下 分布，并且组件会填满整个格子

  • 表格布局还可以在行列参数之后增加两个调整表格与表格之间的间距的参数

  • 通过以下代码实现一个表格布局的窗口

    public class TestLayout {
    public static void main(String[] args) {
        Frame frame = new Frame();

        Button button1 = new Button("1");
        Button button2 = new Button("2");
        Button button3 = new Button("3");
        Button button4 = new Button("4");
        Button button5 = new Button("5");
        Button button6 = new Button("6");

        // 设置表格布局
        frame.setLayout(new GridLayout(2,3));

        frame.add(button1);
        frame.add(button2);
        frame.add(button3);
        frame.add(button4);
        frame.add(button5);
        frame.add(button6);

        frame.setSize(200, 200);
        frame.setVisible(true);
    }
}

动作事件监听

事件监听是指一直监听某个绑定的组件，一旦组件发生事件，则执行动作

  • 下面代码实现点击一下按钮，控制台输出一次”事件触发”： 
    public class TestActionEvent {
    public static void main(String[] args) {
        // 按下按钮，触发事件
        Frame frame = new Frame();

        Button button = new Button();
        frame.add(button);
        frame.setSize(200, 200);
        frame.setVisible(true);

        button.addActionListener(new MyListener());
    }
}

class MyListener implements ActionListener {

    @Override
    public void actionPerformed(ActionEvent e) {
        System.out.println("事件触发");
    }
}

Lambda表达式：当这一个监听器只用一次时，可以使用Lambda表达式创建匿名内部类

button.addActionListener((e -> {
        System.out.println();
    }));

封装：同样，当同一个监听器要对同一种组件用很多次，并且事件的处理方式都一样时，可以将增加监听器和要执行的动作写在一个方法里，以关闭窗口为例：

public class TestActionEvent {
    public static void main(String[] args) {
        // 按下按钮，触发事件
        Frame frame = new Frame();

        windowClosing(frame);
        frame.setSize(200, 200);
        frame.setVisible(true);
    }

    // 窗体关闭事件
    private static void windowClosing(Frame frame) {
        frame.addWindowListener(new WindowAdapter() {
            @Override
            public void windowClosing(WindowEvent e) {
                System.exit(0);
            }
        });
    }
}

多组件监听：要实现一个监听器监听多个组件并做出不同动作，有两种解决方法：

  1. 当含有组件的类本身实现了监听接口时，可以在重写方法时使用e.getSource()获取事件源并对比组件对象

    if (e.getSource() == button) {}

  2. 当监听器脱离于含有组件的类时，监听器内的方法获取不到组件的对象，因此不能使用e.getSource()，需要使用e.getActionCommand()获取触发事件返回的指令

    • 返回指令：按钮返回指令存在一个预设值，即按钮上显示的内容，可以通过调用组件的方法setActionCommand();设置每个组件的返回指令
    start.setActionCommand("");

if (e.getActionCommand() == "") {}

文本框（TextField）：文本框可以让用户输入一行字符，文本框有以下特性

  1. 仅允许输入一行字符
  2. 在文本框按下enter键触发ActionEvent

  • 下面代码实现一个简单的文本框：

    class MyFrame extends Frame {
    public MyFrame() {
        TextField textField = new TextField();
        this.add(textField);

        setVisible(true);
        setSize(200, 200);
    }
}

    在事件触发方法对事件源进行处理：通过e.getSource()可以获得事件源对象，从而直接对事件源进行处理

    • 下面代码实现对文本框事件监听并且执行相应的动作： 
      class MyListener implements ActionListener {

    // 在文本框按下回车触发事件
    @Override
    public void actionPerformed(ActionEvent e) {
        // 获得事件源
        TextField field = (TextField)e.getSource();
        // 获得输入框文本
        System.out.println(field.getText());
        // 清空文本域
        field.setText("");
    }
}

    文本框字符替换：使用文本框对象的setEchoChar('')方法可以使输入的字符表面上替换为指定的字符（如 使用到密码的输入上）

实现一个加法窗口

使用文本框实现一个计算两数之和的窗口

  1. 先写出能运行的程序

    1. 下面的代码实现一个基本的框架
    public class CounterOld extends Frame {
    public static void main(String[] args) {
        new CounterOld();
    }

    public CounterOld() {
        // 组件
        TextField num1 = new TextField(10);
        Label plus = new Label("+");
        TextField num2 = new TextField(10);
        Button count = new Button("=");
        TextField value = new TextField(10);

        // 拼装&布局设置
        setVisible(true);
        setLayout(new FlowLayout());

        add(num1);
        add(plus);
        add(num2);
        add(count);
        add(value);
        pack();
    }
}
    1. 事件处理
      • 问题：事件处理的类访问不到组件
      • 解决：
        1. 提升组件的作用域
        2. 组合思想：在事件处理类中组合另外一个类（使CounterOld对象存在于事件处理类中，并在事件处理类的构造方法中作为参数传入）
          • 关于组合思想：继承和组合都是服用代码的方式，继承是白盒复用，父类代码暴露，破坏了封装性，组合是黑盒复用，被组合进来的对象只能调用其公开属性和方法，更符合面向对象思想
            class OldListener implements ActionListener {
CounterOld counterOld = null;

public OldListener(CounterOld counterOld) {
    this.counterOld = counterOld;
}

@Override
public void actionPerformed(ActionEvent e) {
    int num1 = Integer.parseInt(counterOld.num1.getText());
    int num2 = Integer.parseInt(counterOld.num2.getText());
    counterOld.value.setText(String.valueOf(num1 + num2));
}
}
            至此，已经实现一个基本的加法计算窗口

  1. 优化思想

    1. 面向对象思想：一个类里面包含属性和方法
      • 尽量把变量（属性）往外提
      • 不使用构造方法构建窗体，使用新增的方法
    2. 封装思想：只使用一次的监听器类可以将其变成私有内部类，或者lambda表达式，这样做也省去了获取不到组件对象的问题
    3. 条理性：构建窗体的方法中，代码块尽量分为下面三部分来写：
      • 组件设置
      • 事件设置
      • 布局设置
    public class Counter {
    public static void main(String[] args) {
        new MyCounter().initCounter();
    }
}

class MyCounter extends Frame {
    // 属性
    private TextField number1, number2, value;

    // 方法
    public void initCounter() {

        // 组件设置
        number1 = new TextField(10);
        Label label = new Label("+");
        number2 = new TextField(10);
        Button count = new Button("=");
        value = new TextField(11);

        // 事件添加
        count.addActionListener(new MyListener());

        // 布局设置
        this.setVisible(true);
        this.setLayout(new FlowLayout());
        add(number1);
        add(label);
        add(number2);
        add(count);
        add(value);
        pack();
    }

    private class MyListener implements ActionListener {
        @Override
        public void actionPerformed(ActionEvent e) {
            int num1 = Integer.parseInt(number1.getText());
            int num2 = Integer.parseInt(number2.getText());
            value.setText(String.valueOf(num1 + num2));
            number1.setText("");
            number2.setText("");
        }
    }
}

画笔

画笔可以在容器内绘画各种形状，基本思路类比微软的画图工具：先设置颜色，再绘画图形

  • 需要注意的是：paint方法只在窗口初始化时执行一次
  • 下面的代码实现将一个空心圆形和一个实心正方形绘画于窗体中
    public class MyPaint {
    public static void main(String[] args) {
        new TestPaint().init();
    }
}

class TestPaint extends Frame {
    public void init() {
        setBounds(200,200,600,500);
        setVisible(true);
    }

    @Override
    public void paint(Graphics g) {
        // 画笔，需要有颜色
        g.setColor(Color.blue);
        // 画画
        // 空心圆
        g.drawOval(100,100,100,100);

        g.setColor(Color.gray);
        g.fillRect(300,300,200,200);

        // 画笔用完，还原到最初颜色
        g.setColor(Color.BLACK);
    }
}

鼠标监听

实现鼠标点击画点

基本思路
  • 下面的代码实现鼠标每点击一次就在当前位置画一个点
    public class DrawPoint extends Frame {
    public static void main(String[] args) {
        new DrawPoint();
    }

    // 用于存储坐标点的集合
    ArrayList<Point> points;

    @Override
    public void paint(Graphics g) {
        // 设置颜色
        g.setColor(Color.blue);
        Iterator<Point> iterator = points.iterator();
        while (iterator.hasNext()) {
            Point point = iterator.next();
            g.fillOval(point.x, point.y, 10, 10);
        }

    }

    public DrawPoint() {
        points = new ArrayList<>();

        // 事件设定
        this.addMouseListener(new MouseAdapter() {
            @Override
            public void mousePressed(MouseEvent e) {
                // 每点击一次，在集合中增加当前鼠标位置
                points.add(e.getPoint());

                // 在事件源（窗体）上重绘
                ((DrawPoint)e.getSource()).repaint();
            }
        });

        // 布局设定
        this.setBounds(200, 200, 400, 400);
        this.setVisible(true);
    }
}

二、Swing

Swing继承于AWT，也是AWT的拓展，相比于AWT，Swing不可以直接在容器上添加组件，他需要在容器的面板上添加，使用容器对象的方法getContentPane()获取容器的面板

同样，背景颜色，布局设置也需要在面板上进行

而窗口大小，窗口初始位置，可见性则可以直接在容器上设置

  • 下面代码实现一个简单的JFrame窗口
    public class MyJFrame {
    public static void main(String[] args) {
        new MyJFrame().init();
    }

    private void init() {
        JFrame jFrame = new JFrame();

        // 组件
        JLabel jLabel = new JLabel("你好", SwingConstants.CENTER);

        Container contentPane = jFrame.getContentPane();
        contentPane.setBackground(Color.blue);
        contentPane.add(jLabel);

        // 布局设定
        jFrame.setBounds(100, 100, 200, 200);
        jFrame.setVisible(true);
    }
}

弹窗

弹窗：弹窗本质上也是一个窗口，也遵循窗口的设定规则

  • 需要注意的是，弹窗的关闭事件不可以被设置为EXIT_ON_CLOSE

  • 下面代码实现点击一个按钮产生一个弹窗

    public class MyJFrame {
    public static void main(String[] args) {
        new MyJFrame().init();
    }

    private void init() {
        JFrame jFrame = new JFrame();
        Container contentPane = jFrame.getContentPane();
        contentPane.setLayout(null);

        // 组件
        JButton jButton = new JButton("产生一个对话框");
        // 设定了绝对定位之后，组件的位置相对于窗口而言设置
        jButton.setBounds(30, 30, 200, 50);

        contentPane.add(jButton);

        // 事件设定
        jFrame.setDefaultCloseOperation(WindowConstants.EXIT_ON_CLOSE);
        jButton.addActionListener(new ActionListener() {
            @Override
            public void actionPerformed(ActionEvent e) {
                // 弹窗
                new MyDialog().dialogInit();
            }
        });

        // 布局设定
        jFrame.setBounds(100, 100, 300, 300);
        jFrame.setVisible(true);
    }
}

class MyDialog {
    public void dialogInit() {
        JDialog jDialog = new JDialog();
        JLabel jLabel = new JLabel("哈哈哈哈");
        Container contentPane = jDialog.getContentPane();
        contentPane.setLayout(null);

        contentPane.add(jLabel);

        jDialog.setDefaultCloseOperation(WindowConstants.DISPOSE_ON_CLOSE);

        jDialog.setBounds(100, 100, 500, 500);
        jLabel.setBounds(50, 50, 100, 100);
        jDialog.setVisible(true);
    }
}

关于Swing的默认关闭事件：在Swing中可以使用容器对象的方法setDefaultCloseOperation()来设置关闭事件，有以下几种事件处理：

  1. DO_NOTHING_ON_CLOSE：关闭按钮无效
  2. HIDE_ON_CLOSE：隐藏窗口（任务栏找不到但是程序不会结束）并且窗口占用的资源不会被释放
  3. DISPOSE_ON_CLOSE：隐藏当前窗口并且释放该窗口所占用的资源
  4. EXIT_ON_CLOSE：结束当前程序

关于绝对定位：将布局设置为null，则为绝对定位，面板内所有元件都必须设置大小，否则就算add了不会显示

  • 此时的定位是相对于面板而言的

图标
要画一个图标，首先要继承Icon类，重写里面的方法

特别要注意下面两个方法的返回值是否为图标宽和高，若返回值为0，则可能出现图标和文字重叠的情况

  • 下面代码实现一个标签（Label）里面含有图标和文字
    public class IconTest implements Icon {
    private int width;
    private int height;

    public static void main(String[] args) {
        new IconTest().init();
    }

    public IconTest() {
    }

    public IconTest(int width, int height) {
        this.width = width;
        this.height = height;
    }

    public void init() {
        JFrame jFrame = new JFrame();
        // 图标可以放在标签上 按钮上
        IconTest iconTest = new IconTest(15,15);
        JLabel label = new JLabel("iconTest", iconTest, SwingConstants.CENTER);

        Container contentPane = jFrame.getContentPane();
        contentPane.add(label);

        jFrame.setDefaultCloseOperation(WindowConstants.EXIT_ON_CLOSE);

        jFrame.setVisible(true);
    }

    @Override
    public void paintIcon(Component c, Graphics g, int x, int y) {
        g.fillOval(x, y, width, height);
    }

    @Override
    public int getIconWidth() {
        return width;
    }

    @Override
    public int getIconHeight() {
        return height;
    }
}

面板

swing中的面板和AWT的面板一样，除了自身要放在内容面板内之外，其他操作和AWT一样，组件可以直接放在JPanel里面

滚动条面板：使用ScrollPane为窗口增加一个滚动条。在构造ScrollPane时将一个组件作为参数传入，则在窗体大小改变时，若该组件大小超过窗体，会出现滚动条

单/复选框

单选框：单选框使用到类JRadioButton

需要注意的是：由于单选框只能选择一个，因此要使用类ButtonGroup对单选框进行分组

  • 下面代码实现三个单选框
    public class RadioButtonTest {
    public static void main(String[] args) {
        new RadioButtonTest().init();
    }

  1. 静态画面绘制
    新建一个类继承Panel，所有的游戏内容在该Panel内绘画，重写paintComponent()方法并调用paintIcon()方法在该面板上的特定位置绘画Icon组件

注意：不要使用paintComponents()方法

public class GamePanel extends JPanel {
    // 定义蛇的数据结构
    int length;
    int[] snakeX = new int[705];
    int[] snakeY = new int[705];

    public GamePanel() {
        init();
    }

    // 定义初始状态
    public void init() {
        length = 3;
        snakeX[0] = 82;snakeY[0] = 105;// 头部
        snakeX[1] = 62;snakeY[1] = 105;// 身体
        snakeX[2] = 42;snakeY[2] = 105;// 身体
    }

    // 绘制面板，游戏中的所有东西都是用这个面板绘画
    @Override
    protected void paintComponent(Graphics g) {
        // 清屏消隐
        super.paintComponent(g);
        setBackground(Color.WHITE);
        // 绘制静态面板——头部广告栏
        Data.header.paintIcon(this, g, 17, 15);
        // 绘制静态面板——游戏区域（初始位置22，105）
        g.fillRect(22, 105, 640, 440);

        // 绘制蛇
        Data.snakeHead.paintIcon(this, g, snakeX[0], snakeY[0]);
        Data.snakeBody.paintIcon(this, g, snakeX[1], snakeY[1]);
        Data.snakeBody.paintIcon(this, g, snakeX[2], snakeY[2]);
    }
}

  2. 动态实现

    监听键盘事件控制游戏启停

    简单，只需要在初始化的时候给面板增加一个键盘监听器即可

    注意：

    1. 默认面板不可以获得焦点，不可以获得焦点的组件不可以用键盘控制
    2. KeyTyped是键入类事件，对于键入类型的事件，getKeyCode方法，始终返回VK_UNDEFINED，此时可以使用getKeyChar判断哪个按键按下

    实现小蛇移动

    定时器：使用到swing包下的类Timer，在构造方法里传入：

     1. 延时时间（ms）
 2. 事件监听器

    并使用Timer对象的start()方法启动定时器，每次到达延时时间时，定时器都会触发一次ActionEvent事件并由传入的监听器接收，执行相应的动作

    定时事件中需要做的事：

    1. 若游戏位于启动状态，将每各身体移动到前一各身体的位置（注意一定要从最后一各开始移动，因为是根据前一格的坐标来移动的）
    2. 头部前进一格
    3. 重绘
    if (isStart) {
    // 身体移动
    for (int i = length - 1; i > 0; i--) {
        snakeX[i] = snakeX[i - 1];
        snakeY[i] = snakeY[i - 1];
    }
    // 头部移动
    snakeX[0] += unitLength;

    repaint();
}

    使用键盘监听改变运动方向：

    • 新建一个用于标志目前移动方向的变量，每次头部移动都进行一次判断

switch (direction) {
    case 'U':

    食物

    利用随机数实现食物生成：使用类Random的对象的nextInt()方法（传入参数以自定义随机数边界）

    // 定义食物
// 游玩区域左边界（上边界）x值 + 单位长度 * 随机数
private void initFood() {
    foodX = 22 + unitLength * random.nextInt(31);
    foodY = 105 + unitLength * random.nextInt(21);
}

    吃食物：每次移动都判断小蛇头部坐标是否与食物坐标重合即可

    • 注意：判断是否吃到食物一定在身体移动前执行，因为身体移动会将坐标数组里的每个记录身体坐标的元素变成他前一个元素的值，因此只需要增加length即可
      // 判断是否吃到食物
if (snakeX[0] == foodX && snakeY[0] == foodY) {
    // 吃到食物身体变长
    length++;
    // 生成新的食物
    initFood();
}

    失败判定

    只需要每次移动都遍历一边坐标数组看看身体坐标有没有和头部坐标重合即可
