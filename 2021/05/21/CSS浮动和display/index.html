CSS浮动和display

行内元素排列的方式

display

  • 使用display可以改变元素的属性（行内元素/块级元素/消失）
  • inline：行内元素
  • block：块级元素
  • inline-block：是块级元素但是内联（通常用于制作导航栏，再使用vertical-align: top;对其顶部，并使用line-height设置行高）
  • none：消失

浮动（float）

  • left:左浮动
  • right:右浮动
  • 浮动可以使元素上移一个图层
  • 浮动的盒子可以向左浮动也可以向右浮动，直到它的外边缘碰到父级元素的范围或另一个浮动的元素为止
  • 浮动的元素会把不浮动的元素推开

清除浮动（clear）

  • left：该元素的左侧不允许有浮动元素，若有，该元素往下走
  • right：该元素的右侧不允许有浮动元素
  • both：该元素的两侧不允许有浮动元素

父级边框塌陷

  • 元素浮动之后，它会脱离父级边框，无论该元素多大，父级边框的宽度和高度不会迁就该元素，如何让该元素保持在父级边框里？
  1. 增加父级元素高度（不建议）
  2. 在所有浮动的元素后面加一个空 的div，将该div的两侧浮动清除，则这个div会被挤到浮动元素的最后，而它自己并不浮动，相当于把父级边框撑大。（代码中应该避免空div）
  3. 不指定父级元素的宽度和高度，对父级使用 overflow: hidden;
  4. 对父级元素增加一个伪类 ::after：（建议使用）
    • ::after用于给选中元素增加一个伪行内元素，作为选中元素的最后一个子元素，通常配合content来增加修饰内容
    • ::before可以用于在选中元素的前面增加一个位行内元素

overflow

  • overflow应被定义于一个决定了宽度和高度的父级元素，它会决定当子元素超出范围时该怎么做
  • hidden：当子元素超出范围，隐藏超出部分
  • scroll：当子元素超出范围，使用滚动条

display与float对比

  • 两者都能让元素排成一行
  • display方向不可控，从左到右
  • float方向可控，但是要解决父级边框塌陷问题
