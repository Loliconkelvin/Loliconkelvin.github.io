Java-IO流

流是内存与存储设备之间传输数据的通道

InputStream（字节输入流父接口）

InputStream是所有字节输入流的父类

InputSream类常用方法


```read()```：从输入流中读取一个字节

```read(byte[] b)```：从输入流中读取一定数量的字节，将其存储在缓冲区数组b中

```read(byte[] b, int off, int len)```：将输入流中的len个字节读入到数组b中

## OutputStream（字节输出流父接口）

OutputStream是所有字节输出流的父类

---
**outputSream类常用方法**

```close()```：关闭流

```flush()```：强制刷新输出流并写出所有缓冲的输出字节

```write(byte[] b)```：将数组b写入到输出流

```write(byte[] b, int off, int len)```：将数组b从偏移量off开始的len个字节写入输出流

```write(int b)```：将指定的字节写入0输出流

## 文件字节流

<span class="red bold">构造方法：</span>

文件输入字节流主要用到两个构造方法（输出流同理）

```public FileInputStream(File file)```：传入一个File类对象

```FileInputStream(String name)```：传入文件路径

文件输出流还有个构造方法

```public FileOutputStream(File file, boolean append)```

```public FileOutputStream(String name, boolean append)```：append设定追加模式，当append为true时，写入的内容会在旧内容之后

---
**文件输入流读入数据的方式**

<span class="red bold">一次读一个字节</span>

使用```read()```返回当前读取的字节值，若读完，返回-1

```Java
public static void main(String[] args) throws IOException {
    FileInputStream fis = new FileInputStream("text.txt");
    int buffer = 0;

    while ((buffer = fis.read()) > 0) {
        System.out.println((char)buffer);
    }

    fis.close();
}

一次读多个字节并存入字节数组

使用read(byte b[])将多个字节读取并存入字节数组直到数组填满，返回这次读取的字节数量

public static void main(String[] args) throws IOException {
    FileInputStream fis = new FileInputStream("text.txt");
    byte[] buffer = new byte[1024];
    int length;

    while ((length = fis.read(buffer)) > 0) {
        System.out.println(new String(buffer, 0, length));
    }
    
    fis.close();
}

文件输出流写文件的方式

使用write(byte b[])将字节数组里的内容全部写到流中

public static void main(String[] args) throws IOException {
    FileOutputStream fos = new FileOutputStream("text.txt");

    String data = "hello world";

    fos.write(data.getBytes());

    fos.close();
}

使用文件输入输出流实现文件的复制

字节流可以复制任何文件

public static void main(String[] args) {
    FileOutputStream fos = null;
    FileInputStream fis = null;

    try {
        fos = new FileOutputStream("JAVA-copy.txt");
        fis = new FileInputStream("JAVA.txt");

        byte[] buffer = new byte[1024];
        int length;

        while ((length = fis.read(buffer)) != -1) {
            fos.write(buffer, 0, length);
        }
    } catch (IOException e) {
        e.printStackTrace();
    } finally {
        try {
            fis.close();
            fos.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

字节缓冲流

  • （通过创建缓冲区，先将磁盘里的数据读入缓冲区，输入/输出流从缓冲区读数据，缓冲区读完则缓冲流再去磁盘读数据）提高IO效率，减少访问磁盘的次数
  • 数据存储在缓冲区中，flush试讲缓冲区的内容写入文件中，也可以直接close

---
**BufferedOutputStream构造方法**

```BufferedOutputStream(OutputStream out)```
```BufferedOutputStream(OutputStream out, int size)```：需要传入一个输出流，也可以传入size以自定义缓冲区大小

```write(byte b[])```：将字节数组里的内容全部写到流中
* <span class="red bold">注意：</span>若字节数组的数据不足以填满缓冲区，则write之后数据还在流中，并不会写入到文件
* 调用flush()方法或close()方法关闭流，都可以将缓冲区内的数据写入文件

```flush()```：强制刷新缓冲区，将缓冲区内的数据写出

## 对象流

与对象流相关的流：ObjectOutputStream ObjectInputStream

* 对象流增强了缓冲区功能
* 增强了读写8种基本数据类型和字符串的功能
* 增强了读写对象的功能

使用流传输对象的过程称为序列化、反序列化

---
**对象流的构造方法**

```ObjectInputStream(InputStream in)```

```ObjectOutputStream(OutputStream out)```

---
**对象流常用方法**

```close()```：关闭流

```flush()```：清空流（对象流没有缓冲区）

```writeObject(Object obj)```：将对象写入到文件中

序列化多个成员可以借助集合

```readObject(Object obj)```：将对象读入到流中

* <span class="red bold">注意1：</span>写入的对象需要标记为“可序列化”，<span class="red">需要实现Serializable接口</span>
* <span class="red bold">注意2：</span>写入的对象的类中的属性的类型也需要实现Serializable接口

---
**serialVersionUID**

IDEA默认设置为：<span class="red">允许可序列化类没有序列化UID</span>，开启序列化UID详见 ```Idea关于代码格式的设定```

<span class="red bold">作用：</span>保证序列化和反序列化的对象是同一个对象

* <span class="red bold">注意：</span>这个值要在序列化（存入文件）之前决定，决定之后不要再改。若序列化之后改掉UID，然后反序列化，序列化前后的对象会被认为是两个不同的对象而报错

---
**不序列化的属性**

1. 静态属性不参与序列化
2. <span class="red">要想某个成员属性不被序列化，在属性声明中添加```transient```</span>

## 常见字符编码

* ISO-8859-1：包括ASCII、西欧、希腊语，泰语、阿拉伯语、希伯来语对应的文字符号（1个字节记录）
* UTF-8：针对Unicode码表的可变长度字符编码
* GB2312：简体中文（1或2或3个字节记录）
* GBK：简体中文、扩充（1或2个字节记录）
* BIG5：繁体中文

```write​(char[] cbuf, int off, int len)```：写一个字符数组从偏移量off开始len个字符

## 文件字符流

<span class="red bold">构造方法：</span>

输出流同理

```FileReader(File file)```

```FileReader(String fileName)```

---
**文件字符流读入数据的方式**

跟字节流一样的

<span class="red bold">读入一个字符</span>

```JAVA
public static void main(String[] args) throws IOException {
    FileReader reader = new FileReader("text.txt");

    int data;
    while ((data = reader.read()) != -1) {
        System.out.print((char)data);
    }

    reader.close();
}

读入多个字符

public static void main(String[] args) throws IOException {
    FileReader fr = new FileReader("text.txt");

    char[] buffer = new char[1024];
    int length;
    while ((length = fr.read(buffer)) != -1) {
        System.out.print(new String(buffer, 0, length));
    }

    fr.close();
}

文件字符输出流写数据的方式

public static void main(String[] args) throws IOException {
    FileWriter fw = new FileWriter("text.txt");

    String data = "你好你好你好";
    fw.write(data);

    fw.close();
}

字符流实现文件复制

注意：字符流只能复制文本文件，图片、声音等二进制文件不能复制

public static void main(String[] args) throws IOException {
    FileReader fr = new FileReader("JAVA.txt");
    FileWriter fw = new FileWriter("JAVA-cp.txt");

    char[] buffer = new char[1024];
    int length;
    while ((length = fr.read(buffer)) != -1) {
        fw.write(buffer, 0, length);
    }

    fw.close();
    fr.close();
}

<span class="red bold">字符缓冲输入流方法：</span>

```close()```：关闭流，<span class="red">关闭缓冲流会顺带将传入的流也关闭</span>

```JAVA
public static void main(String[] args) throws IOException {
    FileReader fr = new FileReader("text.txt");
    BufferedReader br = new BufferedReader(fr);

    String data;

    while ((data = br.readLine()) != null) {
        System.out.println(data);
    }

    br.close();
    fr.close();
}

字符缓冲输出流方法：

除了write()写出方法外，字符缓冲流还提供了以下方法：


```JAVA
public static void main(String[] args) throws IOException {
    FileWriter fw = new FileWriter("text.txt");
    BufferedWriter bw = new BufferedWriter(fw);

    String data = "这是一段文本";

    for (int i = 0; i < 10; i++) {
        bw.write(data);
        bw.newLine();
        bw.flush();
    }

    bw.close();
}

## File类

File类是一个 文件 或 文件夹 的抽象表示

---
**分隔符**

File类内置两种常量分隔符
1. 路径分隔符（pathSeparator）：```;```
2. 名称分隔符（separator）：```\```

---
**文件操作**

1. 创建文件
    
    <span class="red">创建File对象并不代表创建文件</span>

    ```exists()```：判断当前文件或目录是否存在

    ```createNewFile()```：根据该File对象的路径创建文件，返回创建成功（true）或失败（false）

    因此创建文件时，可以这样：
    ```JAVA
    public static void main(String[] args) throws IOException {
        File file = new File("D:\\test.txt");

        if (!file.exists()) {
            boolean b = file.createNewFile();
            System.out.println("创建结果" + b);
        }
    }

  1. 删除文件

    1. 直接删除
    
    2. 在JVM退出时删除

    ```deleteOnExit()```：根据File对象的路径在JVM退出时删除文件

3. 获取文件信息

    ```getAbsolutePath()```：获取文件的绝对路径

    ```getPath()```：获取相对路径

    ```getName()```：获取文件名

    ```getParent()```：获取父目录路径

    ```length()```：获取文件字节数

    ```lastModified()```：获取文件最后修改日期，返回毫秒数，使用Date类构造函数接收

    ```canWrite()```：判断文件是否可写

    ```isFile()```：判断该对象是否为一个文件

    ```isHidden()```：判断该文件是否隐藏

---
**文件夹操作** 

1. 创建文件夹

    ```mkdir()```：创建文件夹，<span class="red">只能创建单级目录</span>

    ```mkdirs()```：创建文件夹

2. 删除文件夹

    1. 直接删除

    ```delete()```：删除文件夹，<span class="red">只会删除最后一级的目录，且只能删除空目录。</span>

    2. 在JVM退出时删除

    ```deleteOnExit()```：退出JVM时删除文件，<span class="red">只会删除最后一级的目录，且只能删除空目录。</span>

3. 获取<span class="red">最后一层</span>文件夹信息

    ```getAbsolutePath()```：获取最后一层文件夹的绝对路径

    ```getPath()```：获取相对路径

    ```getName()```：获取最后一层文件夹名

    ```getParent()```：获取最后一层文件夹的父目录路径

    ```lastModified()```：获取最后一层文件夹的最后修改时间

4. 判断

    ```isDirectory()```：判断该对象是否为一个文件夹

    ```isHidden()```：判断该文件夹是否隐藏

5. 遍历

    ```list()```：获取该目录下的所有 <span class="red">文件名 和 文件夹名</span>，返回String[]

    ```listFiles()```：获取该目录下的所有 <span class="red">文件 和 文件夹 对应的File类对象</span>

    ```listFiles(FileFilter filter)```：传入文件过滤器作为参数

---
**实现递归遍历文件**

```JAVA
public static void main(String[] args) {
  File dir = new File("F:\\LostLife_1.4_unc");

  show(dir);
}

public static void show(File dir) {
  File[] files = dir.listFiles();

  if (files != null) {
      for (File file : files) {
          System.out.println(file.getName());
          // 若为文件夹，输出其里面的文件
          if (file.isDirectory()) {
              show(file);
          }
      } 
  }
}

实现递归删除文件

public static void main(String[] args) {
  File dir = new File("E:\\LostLife_1.4_unc");

  delete(dir);
}

public static void delete(File dir) {
  File[] files = dir.listFiles();

  if (files != null) {
      for (File file : files) {
          // 若为文件夹，递归
          if (file.isDirectory()) {
              delete(file);
          // 若为文件，删除
          } else {
              file.delete();
          }
      }
  }
  // 删除传入的文件夹
  dir.delete();
}

Filefilter文件过滤器接口

当调用listFile()方法时，支持传入Filefilter接口的实现类对获取的文件进行过滤，只有当满足条件的文件才可出现在返回值中

// 过滤器内的方法传入一个File类型参数
File[] files = dir.listFiles((pathname) -> {
   if (pathname.getName().endsWith(".jpg")) {
       return true;
   } else {
       return false;
   }
});
  • 省略写法：
File[] files = dir.listFiles(pathname -> pathname.getName().endsWith(".jpg"));
