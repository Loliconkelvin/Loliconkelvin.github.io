1-1 GPIO

1-1-1 LED闪烁（单IO口选择与控制）

  1. 使能IO口时钟

    • 找到RCC_APB2时钟控制函数（stm32f10x_rcc.h拉到最下面，找693行RCC_APB2PeriphClockCmd）并使用
    • 右键函数，Go to the definition of the…… 跳转到函数定义
    • 往上滚动，选择外设（如点亮PA0口则需要使能GPIOA，选择RCC_APB2Periph_GPIOA），放到第一个参数
    • 第二个参数也可以在定义上方找到，选择Enable，放到第二个参数

  2. 初始化GPIO

    • 找到GPIO_init函数（stm32f10x_gpio.h拉到最下面，找351行GPIO_Init）并使用
    • 右键函数跳转到定义
    • 第一个参数是GPIOx，x可以是A-G
    • 第二个参数是一个结构体，先把结构体数据类型复制，粘贴指代码（注意不要把指针的星号复制过去）
    • 定义一个名为GPIO_InitStructure的结构体，将其地址作为GPIO_init的第二个参数（取地址符&）
      • 用“.”引出结构体成员 GPIO_Mode GPIO_Pin GPIO_Speed
    • 右键成员跳转到定义，可以看到成员右边的注释：该成员可以是 xxxx_typedef 中的一个值，选中 xxxx_typedef 按 ctrl + f 搜索 -> find next
      • 若找到多个定义，选择 class为menber 的结果
    • 上滚找到枚举，复制其中一个作为成员的值

  • GPIO_Mode

    成员值 模式 说明
    GPIO_Mode_AIN 模拟输入（Analog In） GPIO无效，引脚接入内部ADC（模数转换器）
    GPIO_Mode_IN_FLOATING 浮空输入 可以读取引脚电平，若引脚浮空，则状态不稳定
    GPIO_Mode_IPD 下拉输入（In Pull Down） 可读取引脚电平，引脚接下拉电阻，浮空则默认低电平
    GPIO_Mode_IPU 上拉输入（In Pull Up） 可读取引脚电平，引脚接上拉电阻，浮空则默认高电平
    GPIO_Mode_Out_OD 开漏输出（Out Open Drain） 可输出引脚电平，高电平为高阻态，低电平接Vss
    GPIO_Mode_Out_PP 推挽输出（Out Push Pull） 可输出引脚电平，高电平接Vdd，低电平接Vss
    GPIO_Mode_AF_OD 复用开漏输出（Atl Open Drian） 由片上外设控制，单片机输出无效，但输入有效,高电平为高阻态，低电平接Vss
    GPIO_Mode_AF_PP 复用推挽输出（Atl Push Pull） 由片上外设控制，单片机输出无效，但输入有效，高电平接Vdd，低电平接Vss

  • 注意：在输出模式下，输入是有效的，也就是说可以输出完之后读一下引脚状态。
    但是在输入模式下，输出是无效的，也就是说输出状态下网引脚上写东西是没用的


开漏输出与推挽输出

开漏输出情况下，P-MOS无效，高输出会使N-MOS断开
这会直接使引脚和单片机断开，高电平没有驱动能力
低电平使N-MOS导通，引脚接Vss

推挽输出情况下，P-MOS和N-MOS均有效
低电平接Vss，高电平接Vdd
高低电平均有驱动能力

  • GPIO_Pin
    对应想要控制的引脚号，或全部引脚


  • GPIO_Speed
    对应速度，一般选择50MHz的

  1. GPIO输出函数

    • GPIO_SetBits(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin);
      可以将指定组的指定引脚设定为高电平
      第一个参数是GPIOx，x可以是A-G
      第二个参数是GPIO_Pin_x，n可以是0-15


    • GPIO_ResetBits(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin);
      可以将指定组的指定引脚设定为低电平


    • GPIO_WriteBit(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin, BitAction BitVal);
      可以将指定组的指定引脚设定为高/低电平（取决于第三个参数）
      第三个参数可以通过赚转到函数定义查看，可以是下面两个值：

      值 电平
      Bit_SET 高电平
      Bit_RESET 低电平

1-1-2 流水灯（多IO口选择与控制）

  • 流水灯相比于LED闪烁，主要区别是要初始化多个IO口，以及要控制多个IO口
  1. 多IO口初始化
    通过看查GPIO_pins_define定义，可以知道，单片机一组16个端口分别对应的是16位数中的一位

    （如GPIO_Pin_7对应0000 0000 1000 0000 即0x0080）
    （↑对应第八位是因为IO口编号和位数都是从0开始数）

  • 所以想要选中多个端口把他们按位或即可

  • 如果要选择一组中的所有16个端口，可以把GPIO_Pin_All赋给GPIO_InitStructure.GPIO_Pin


  • 同样可以通过按位或去的多个端口的还有：
    RCC_APB2PeriphClockCmd 的第一个参数 RCC_APB2Periph
    GPIO_SetBits 的第二个参数 GPIO_Pin
    GPIO_ResetBits 的第二个参数 GPIO_Pin

  1. 多IO口控制

    • 使用GPIO_Write函数进行多IO口控制
    • 第一个参数为GPIOx
    • 第二个参数为一个16位的数值，该值会直接写到GPIO_ODR寄存器，相当于一个函数控制一组IO口

  2. 循环逻辑

    1. 无脑写数(最无脑 最安全 不担心出错) 
      while(1){
    GPIO_Write(GPIOA, ~0x0001);
    Delay_ms(500);
    GPIO_Write(GPIOA, ~0x0002);
    Delay_ms(500);
    GPIO_Write(GPIOA, ~0x0004);
    Delay_ms(500);
    GPIO_Write(GPIOA, ~0x0008);
    Delay_ms(500);
    GPIO_Write(GPIOA, ~0x0010);
    Delay_ms(500);
    GPIO_Write(GPIOA, ~0x0020);
    Delay_ms(500);
    GPIO_Write(GPIOA, ~0x0040);
    Delay_ms(500);
    GPIO_Write(GPIOA, ~0x0080);
    Delay_ms(500);
}
    2. 数组写数（比较无脑 for循环变量要考虑） 
      char gpioValue[8] = {
    0x0001, 0x0002, 0x0004, 0x0008,
    0x0010, 0x0020, 0x0040, 0x0080,
};

while(1){
    for(char i = 0; i < 8; i++){
        GPIO_Write(GPIOA, ~gpioValue[i]);
        Delay_ms(500);
    }
}
    3. 循环写数（需要点逻辑 for循环变量要考虑） 
      while(1){
    for(char i = 0; i < 8; i++){
        GPIO_Write(GPIOA, ~(0x0001 << i));
        Delay_ms(500);
    }
}
    • 注意点灯是低电平，最后别忘记取反

1-1-3 有源蜂鸣器（简单上手）

  • 给PB12输出低电平

  • 完整代码：

    #include "stm32f10x.h"                  // Device header
#include "Delay.h"

int main(void){
    //使能时钟    
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE);

    //初始化IO口    
    GPIO_InitTypeDef GPIO_InitStructure;
    GPIO_InitStructure.GPIO_Mode =GPIO_Mode_Out_PP;
    GPIO_InitStructure.GPIO_Pin =GPIO_Pin_12;
    GPIO_InitStructure.GPIO_Speed =GPIO_Speed_50MHz;
    GPIO_Init(GPIOB, &GPIO_InitStructure);

    //写IO口    
    while(1){
        GPIO_WriteBit(GPIOB, GPIO_Pin_12,Bit_RESET);
        Delay_ms(500);
        GPIO_WriteBit(GPIOB, GPIO_Pin_12,Bit_SET);
        Delay_ms(500);
    }
}

回顾C语言——结构体的定义

  • 使用结构体可以组合不同数据类型的数据

  • 结构体的定义：

    struct Books {
    char title[20];
    char author[50];
    int id;
} book;

  • 其中：

    1. Books：结构体标签，可以看成是结构体名，但不是严格的结构体名，有调用的要求
    2. char tile[20]等: 结构体包含的变量，称结构体成员
    3. book：结构体变量，结构体可以看成一种数据类型，他当然可以有变量，这里的book是一个结构体变量，他里面含有title，author，id三个参数

  • 规定：

    1. 结构体标签和结构体变量时可选的，但一般至少要有一个，除非……（见struct与typedef连用）
      • 只有标签：当你不想这么快新建一个结构体变量时，可以只写标签
      • 只有结构体变量：当你只想用这个结构体创建唯一的一个变量的时候，可以只写结构体变量
      • 我全都要：当你又想多次用结构体新建变量，邮箱马上新建一个变量的时候，可以两个都写
    • tips：当定义结构体的时候，你可以创建多个结构体变量，逗号隔开。 
      struct Books{
    char title[50];
    int id;
} book1, *pBook2;

    1. 使用结构体标签的结构体变量定义格式： 
      // 要使用struct + 结构体标签 + 结构体变量名 的格式
struct Books book1;
struct Books* pBook2;

  • 一般，我们会把struct和typedef连用形成以下格式，以规避还要使用 struct + 标签 这样繁琐的写法来定义结构体变量：

    typedef struct {
    char title[20];
    int id;
} Book;

// 这样我们就可以用变量类型来定义变量了
Book book1;
Book* pBook2;
    • 注意：这里的Book不再是结构体变量，而是变量类型的命名，Book类型就是这个结构体的类型
    • 在单片机这边，一般会命名为 StructName_t StructName自己起，反正 _t 就对了- -

回顾C语言——结构体成员的调用（普通变量和指针变量）

  • 与Java的非静态变量一样，都是用点来调用

    // 使用 结构体变量名. 结构体成员名 类调用成员
book.id = 10;

  • 如果你定义了一个指针结构体变量，要使用->来调用

    // pBook是一个指针变量
pBook2->id = 10;

回顾C语言——枚举

  • 定义枚举变量以使该变量的取值受限

  • 枚举的定义：

    // 枚举的定义
// 变量week只能取枚举里面的值
enum WEEK {
    MON = 1,
    TUE,
    WED,
    THU,
    FRI,
    SAT,
    SUN
} week;

// 变量的定义
enum WEEK week = WED;

  • 和结构体一样，WEEK枚举名，week枚举变量就不用说了，这样定义的时候两个必须至少有一个。


  • 注意：

    1. 枚举里的值要用逗号隔开
    2. 如果枚举里面的值对应的数是+1递增的，则只需要写第一个的数值，如果都不写数值，则默认从0开始+1
    3. 给枚举变量赋值的时候，只能赋值字符串，若要赋值其对应的数值，需要进行类型转换 
      WEEK week = (WEEK)1;

  • 当然也可以使用typedef

    // 使用typedef的枚举定义
// 变量week只能取枚举里面的值
typedef enum {
    MON = 1,
    TUE,
    WED,
    THU,
    FRI,
    SAT,
    SUN
} WEEK;

// 使用typedef之后的变量定义
WEEK week = MON;
