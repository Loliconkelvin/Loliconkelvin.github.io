Java网络编程

发表于 2021-11-07 更新于 2022-02-28 分类于 Java

初识网络编程

初识通信协议（TCP与UDP）

  • TCP：形如打电话，双方必须建立连接，才能实现交流
  • UDP：形如发短信，双方不需要建立连接，发送方只管发送，可能出现接收方收不到的情况

网络通信要素

  1. 通信双方的地址：

    • IP
    • 端口号

  2. 通信双方需要建立通信规则（通信协议）：

    • 如http ftp smtp tcp udp等

  3. Java在网络通信中：

    • 万物皆对象，有要素就肯定有其对应的类，IP类、通信协议类等

IP地址类（InetAddress）

  • IP地址的用途：IP地址具有唯一性，用于定位网络上的一台或多台计算机


  • 一个特殊的IP地址：127.0.0.1 localhost 指本机


  • IP地址的分类：（两种分类方式）

    1. 通过IP地址分类

      • IPV4：由四个字节组成（每个数字可取0~255）共32位，形如 127.0.0.1
      • IPV6：由八个16位无符号整数组成（每组数字可取0000~FFFF），共128位，形如 6562:8652:0aff:91cb:3d4e:1307:0000:abcd

    2. 通过公/私网分类
      公网：互联网，还下分ABCD类地址，他们的网段

      私网：局域网（192.168.xxx.xxx）


  • 域名

    • 域名的意义：为解决IP地址难记的问题，相当于给IP地址起一个名字

IP套接字地址类（InetSocketAddress）

  • 端口
    • 表示计算机上的一个程序的进程
    • 端口号被规定取值为0~65535
    • TCP协议与UDP协议各有65536个端口号可供分配，允许有程序使用不同协议但端口号相同，但单个协议下端口号不可冲突
    • 端口分类：
      • 公有端口：（0~1023）（以下是默认端口）

        用于内置进程或服务器，最好不要占用

        • HTTP：80
        • HTTPS：443
        • FTP：21
        • Telent：23
      • 程序注册端口：（1024~49151）（以下是默认端口）

        用于给程序和用户使用

        • Tomcat：8080
        • MySQL：3306
        • Oracle：1521
      • 动态端口/私有端口：（49152~65535）

通信协议

  • TCP/IP协议层

    1. TCP：用户传输协议

      • 连接
      • 三次握手 四次挥手

        A:在？
        B:在！
        A:连！

      A:走了
      B:？
      B:确定？
      A:确定！

      • 客户端与服务端连接
      • 传输完成就释放连接，效率低（射完就跑（bushi

    2. UDP：用户数据报协议

      • 不连接
      • 客户端与服务端没有明确界限
      • 不管接收方有没有准备好接收都会发送

TCP

TCP实现客户端——服务端发送信息

客户端

  • 服务端需要负责：
    1. 连接客户端（Socket）
      • 知道服务端的IP地址和端口
    2. 发送信息
      // 客户端
public class TcpClient {
    public static void main(String[] args) {
        OutputStream os = null;
        Socket socket = null;

        // 需要知道服务端地址
        // 需要连接服务端
        try {
            InetAddress server = InetAddress.getByName("127.0.0.1");

            // 需要端口号
            int port = 1145;

            // 创建socket连接
            socket = new Socket(server, port
            );

            // 发送消息 （IO流
            os = socket.getOutputStream();

            os.write("牛哇".getBytes());

        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            // 关闭服务
            if (os != null) {
                try {
                    os.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (socket != null) {
                try {
                    socket.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

服务端

  • 客户端需要负责：
    • 拥有自己的IP地址和端口
    1. 等待客户端连接（ServerSocket）
    2. 接收信息
    3. 使用管道流处理并输出信息
      public class TcpServer {
    public static void main(String[] args) {
        ServerSocket serverSocket = null;
        Socket socket = null;
        InputStream is = null;
        ByteArrayOutputStream baos = null;

        // 需要有一个地址
        try {
            serverSocket = new ServerSocket(1145);
            // 等待客户端连接
            socket = serverSocket.accept();
            // 读取客户端消息
            is = socket.getInputStream();

            /*
            // 输出（String接收）
            byte[] buffer = new byte[1024];
            int length;
            while ((length = is.read(buffer)) != -1) {
                String msg = new String(buffer, 0, length);
                System.out.println(msg);
            }
            */

            // 输出（流接收）
            baos = new ByteArrayOutputStream();
            byte[] buffer = new byte[1024];
            int length;
            while ((length = is.read(buffer)) != -1) {
                baos.write(buffer, 0, length);
                System.out.println(baos);
            }
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            // 关闭流和连接
            if (baos != null) {
                try {
                    baos.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (is != null) {
                try {
                    is.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (socket != null) {
                try {
                    socket.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (serverSocket != null) {
                try {
                    serverSocket.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

TCP实现客户端——服务端发送文件

客户端

  • 思路和发送信息一样，客户端需要完成的：
    1. 需要知道服务端IP和端口（Socket）
    2. 发送文件
      • 将文件传入流
        • 实现 文件 -> 文件输入流 -> 输出流 的转换
          public class TcpFileClient {
    public static void main(String[] args) {
        OutputStream os = null;
        FileInputStream fis = null;

        // 连接服务端
        int port = 4433;
        try {
            Socket socket = new Socket("localhost", port);
            // 定位文件
            File file = new File("test.txt");
            // 准备输出流
            os = socket.getOutputStream();
            // 设置文件输入流(可以直接使用文件名而非File对象)
            fis = new FileInputStream(file);

            // 将文件写入缓冲区
            byte[] buffer = new byte[1024];
            int length;
            while ((length = fis.read(buffer)) != -1) {
                // 从缓冲区提取到输出流
                os.write(buffer, 0, length);
            }

        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            // 关闭服务
            if (fis != null) {
                try {
                    fis.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (os != null) {
                try {
                    os.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }

    }
}

服务端

  • 服务端需要做的事：
    1. 拥有自己的IP和端口（SocketServer）
    2. 监听客户端接入
    3. 接收文件（接收什么就用什么管道流）
      • 实现 输入流 -> 文件输出流 -> 文件 的转换
        public class TcpFileServer {
    public static void main(String[] args) {
        ServerSocket server = null;
        Socket client = null;
        InputStream is = null;
        FileOutputStream fos = null;
        // 拥有自己的端口
        InetSocketAddress port = new InetSocketAddress(4433);
        // 开启服务器
        try {
            server = new ServerSocket(4433);
            // 等待客户端接入
            client = server.accept();
            // 接收客户端文件
            is = client.getInputStream();
            // 将文件写入缓冲区
            fos = new FileOutputStream("test_re.txt");
            byte[] buffer = new byte[1024];
            int length;
            while ((length = is.read(buffer)) != -1) {
                // 将文件写到本地
                fos.write(buffer, 0, length);
            }
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            // 关闭服务
            if (fos != null) {
                try {
                    fos.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (is != null) {
                try {
                    is.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (client != null) {
                try {
                    client.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (server != null) {
                try {
                    server.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

拓展——若多次传输

  • 下面是一段有服务端确认传输完毕的TCP文件传输

    客户端

    public class TcpFileClient {
    public static void main(String[] args) {
        OutputStream os = null;
        FileInputStream fis = null;

        // 连接服务端
        int port = 4433;
        try {
            Socket socket = new Socket("localhost", port);
            // 定位文件
            File file = new File("test.txt");
            // 准备输出流
            os = socket.getOutputStream();
            // 设置文件输入流(可以直接使用文件名而非File对象)
            fis = new FileInputStream(file);

            // 将文件写入缓冲区
            byte[] buffer = new byte[1024];
            int length;
            while ((length = fis.read(buffer)) != -1) {
                // 从缓冲区提取到输出流
                os.write(buffer, 0, length);
            }

            // 传输结束，不关闭可能会两边都不会结束
            socket.shutdownOutput();

            // 接收服务端的确认信息，确认接收完毕
            InputStream is = socket.getInputStream();
            ByteArrayOutputStream baos = new ByteArrayOutputStream();
            while ((length = is.read(buffer)) != -1) {
                baos.write(buffer, 0, length);
            }
            System.out.println(baos.toString());
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

服务端

public class TcpFileServer {
    public static void main(String[] args) {
        ServerSocket server = null;
        Socket client = null;
        InputStream is = null;
        FileOutputStream fos = null;
        // 拥有自己的端口
        InetSocketAddress port = new InetSocketAddress(4433);
        // 开启服务器
        try {
            server = new ServerSocket(4433);
            // 等待客户端接入
            client = server.accept();
            // 接收客户端文件
            is = client.getInputStream();
            // 将文件写入缓冲区
            fos = new FileOutputStream("test_re.txt");
            byte[] buffer = new byte[1024];
            int length;
            while ((length = is.read(buffer)) != -1) {
                // 将文件写到本地
                fos.write(buffer, 0, length);
            }

            // 告诉客户端接收完毕
            OutputStream os = client.getOutputStream();
            os.write("接收完毕".getBytes());
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            if (fos != null) {
                try {
                    fos.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (is != null) {
                try {
                    is.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (client != null) {
                try {
                    client.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (server != null) {
                try {
                    server.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

解析

  • 要点：
    1. 接收什么内容就用什么流接收
    2. 若客户端/服务端在socket的流中传输完没有直接执行下面的close关闭流而是还有事要干，这时候接收方会因为流还存在而一直等待传输方，因此需要传输方调用shutdownOutput()关闭传输
