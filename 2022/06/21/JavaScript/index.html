JS 基本格式

JS 代码与 CSS 代码都可以独立分出去一个文件，也可以直接写在 HTML 文件内

  • 若要写在 HTML 文件内，JS 代码需要可以放在 <head> 标签或 <body> 标签内，不过通常是放在 <body> 标签内最后，使用 <script> 标签包裹

    <html lang="zh-Hans">
  <head>
    <meta charset="UTF-8" />
    <title>Document</title>
  </head>

  <body>
    <script>
      alert('hello world');
    </script>
  </body>
</html>

  • 若要写成一个独立文件，则在 HTML 内使用 <script> 标签并且指定参数 src 进行进入。JS 文件内直接写 JS 语句，不需要额外的标签包围

    注意：<script></script> 务必成对出现

    <html lang="zh-Hans">
  <head>
    <meta charset="UTF-8" />
    <title>Document</title>
  </head>

  <body>
    <script src="JS/script.js"></script>
  </body>
</html>
    alert('hello world');

JavaScript 变量与常量

变量

变量有两种声明方式，让我们看看他们的区别，在这里也能看到 JS 从乱玩到有些严谨的过程

规范：通常我们会把要使用的变量在一开始就定义好，然后再一个个赋值，或者在一开始就赋值，通常不会在中途定义变量

// var
var num_var = 1;

// let
let num_let = 1;

但在此之前，我们先看看这里会出现的几个作用域：

作用域：

  1. window(Global)：全局变量，在该 HTML 读取到的 JS 中通用（极不建议跨 JS 使用变量）
  2. Script：大块，为了将属性与全局变量分离，在该 HTML 读取到的 JS 中通用（极不建议跨 JS 使用变量）
  3. window(Local)：局部，在该函数中通用
  4. Block：小块，在该代码块中通用

var：

在 ES5 及以前，声明变量只能使用 var 定义，并且都为全局斌量，并且每个全局变量和全局方法都会挂载到 window 下（相当于 Java 的 Object 类，所有类都会继承它）

并且它还有几个离谱的特性：

  • 可以在变量定义之前访问

  • 允许重复定义，后一次定义会覆盖前一次定义

  • 可以突破代码块被访问（如 for 循环等），因为在 ES5 之前，没有 Block 作用域，在代码块内使用 var 声明的变量，依然放在 window(Global) 下

  • 不可以突破函数作用域，因为在 ES5 之前，就已经有 window(Local) 作用域。一个在全局下定义的函数被认为是继承了 window(Global) 的一个子类 window(Local)，它自己有一个 window 用于存放该函数内声明的变量和函数，，函数走完则该 window(Local) 被清除，在里面的变量也就没了

  • 可以被同级别或更低级的函数块访问

let：

在 ES6 时，增加了 let 和 const 分别是多了一种定义变量的方式和定义常量。
ES6 有个重要的改变就是将全局对象的属性（也就是在 window 里面声明的属性和函数）和全局变量分离，let 就用来做这一点。

并且它与 var 背道而驰，因为：

  • let定义的变量禁止在定义之前访问

  • 禁止重复定义

  • 不可以突破函数作用域访问，在函数内使用 let 定义的变量依然是放在 window(Local) 内，函数走完则该 window(Local) 被清除，在里面的变量也就没了

  • 不可突破代码块被访问（如 for 循环等），因为在 ES6，在代码块内用 let 声明的变量，会被放在 Block 作用域，用完马上清除

  • 可以被同级别或更低级的函数块访问

函数内外的变量冲突问题：

若内部函数有变量与外部函数同名，即使两个都是用 let 声明，它们因为作用域完全不同而视为不同的变量。

就近原则：函数要使用内外同名的变量时，会先从函数内查找

全局变量冲突问题：

这也是建议使用 let 的原因，因为 let 是不允许重复定义的，这也是为什么上面 window(Global) 和 Script 写了 “在该 HTML 读取到的 JS 中通用”。

  • 当一个使用 let 定义的变量在其他 JS 文件中被重新声明，会报错

  • 但是当一个使用 var 定义的变量在其他 JS 文件中被重新声明，它的值会被更改，这会造成该变量的值会同时被几个 JS 文件更改，导致该变量的值模糊。

解决：如果非要使用 var 可以使用将该变量使用一个全局对象包装的方式（没有什么是套一层不能解决的）解决：

var global_container = {

}

global_container.name = '张三';
global_container.add = function(a, b) {
  return a + b;
}

JavaScript 流程控制

判断

都是学过的东西

// if
if (条件语句) {
  代码块;
}

// if ...else
if (条件语句) {
  代码块;
} else {
  代码块;
}

// if ...else if ...else
if (条件语句) {
  代码块;
} else if (条件语句) {
  代码块;
} else {
  代码块;
}

循环

// for循环
for(let i = 0; i < 10; i++) {
  console.log(i);
}

// while循环
whlie(i < 10) {
  i++;
  console.log(i);
}

do {
  i++;
  console.log(i);
} while(i < 10);

// 增强for（普通1）
// 使用 of 将arr中的每个元素赋给object
for(let object of arr) {
  console.log(object);
}

// // 增强for（普通1）
// 使用 in 将arr中的每个元素 的下标 赋给object

for(let object in arr) {
  console.log(object);
}

// 增强for（流形式）
arr.foreach(function(value) {
  console.log(value);
});

数组

JavaScript 中的数组什么都可以放，数组公交车！

它 TMD 还可以自由伸缩（详见数组方法）

并且当访问的下标超出数组最大下标时，只会返回 undefined

// 建议使用这种方式
var arr = [1, 1.2, null, true];

// 这个明显就是从Java抄过来的
new Array(1, 2.3, null);

// 取值
console.log(arr);
console.log(arr[2]);

// 赋值
arr[0] = 5;

数组常用属性和方法：

length：（属性）长度

  • 注意：若给长度赋值，可以实现数组长度变更。若值大于原长度，后面的 n 个值为 undefined。若值小于原长度，会有数据丢失

indexOf(Object o);：通过元素获得下标信息（元素是严格区分字符串和数字的）
slice(int n)：去除 Array 的前 n 个元素，返回处理后的数组
slice(int from, int to)：截断数组，结果包含原数组下标为 [from,to) 的位置的元素
push(Object o, ...)：将传入的元素加入到数组最后，加入后顺序和传入顺序一样（传入 a, b 加入后：…, a, b）
pop()：将数组最后一个元素弹出，并返回该元素
shift(Object o, ...)：将传入的元素加入到数组最前，加入后顺序和传入顺序一样（传入 a, b 加入后：a, b, …）
unshift()：将数组最前一个元素弹出，并返回该元素
sort()：排序
reverse()：反转
concat(Array arr)：将传入的数组拼接到原数组后面，并返回拼接后数组
join(String s)：使用传入的字符串连接数组的每个元素，返回连接后字符串

对象

JavaScript 甚至可以凭空搞出来一个对象，不需要类，不需要 new！

同样使用 var 来定义，使用大括号包围，使用点访问内部参数，熟悉的味道！

其本质是若干个键值对，其中键为字符串

var person = {
  name: '张三',
  age: 23,
  tags: ['js', 'html']
};

// 取值
console.log(person);
console.log(person.name);
console.log(person['name']);

// 赋值
person.name = 值;

注意：取一个不存在的属性并不会导致报错，只会返回 undefined

离谱！动态增减对象属性：

哪门编程语言可以做到在对象定义完之后还能删除对象的属性……

delete 对象名.属性名实现删除该对象的指定属性

对象名.新的属性名 = 值;给一个虚空属性赋值就可以实现给该对象增加一个属性并且赋值

常用方法？

'属性名' in 对象名：返回该属性（或叫这个名字的方法）是否存在于指定对象中

hasOwnProperty(String s)：返回该属性（或叫这个名字的方法）是否存在于指定对象中（仅自身拥有的属性和方法，排除继承的）

Map 和 Set （ES6 新特性）

Map：

使用以下代码创建一个 Map 键值对（其实是二维数组）

var map = new Map([['key1', value1], ['key2', value2], ...]);

get(String key)：根据传入的键，返回其对应的值，找不到返回 undefined

set(String key, Object o)：若键值不存在于 map 中，新建一个键值对并应用传入的值。若存在，修改它的值

delete(String key)：删除传入的键所对应的键值对

Set：

使用以下代码创建一个 Set 集合，注意 Set 集合自动去重

var set = new Set([1, 2, 3]);

add(Object o)：在集合最后增加传入的元素（若原来就有则添加失败）

delete(Object o)：删除集合中为该值的元素

has(Object o)：返回集合中是否含有该元素

for 遍历方法：

Map 和 Set 都可以使用 for ...of 的方式进行遍历

for (let value of map) {
  console.log(value);
}

Iterator 迭代器：

Map 和 Set 都内置了 Iterator 迭代器，通过以下代码显式定义迭代器：

var map = new Map([['1', 10],['2', 20],['3', 30]]);

// 定义迭代器
var iterator = map[Symbol.iterator]();

// 定义值接收用变量
var key_value = iterator.next();

whlie(!key_value.done) {
  console.log(key_value);
  key_value = iterator.next();
}

严格检查模式

严格检查模式用于不让你利用 JavaScript 的随意性玩得这么嗨，比如说变量一定要用 var 或 let 定义，禁止：

i = 1;

要开启严格检查模式，只需要在 JavaScript 代码第一行加上 'use strict';

严格检查模式开启之后，在页面加载完后，若有触发严格监察模式的错误的代码，会在控制台报错（当然 IDEA 这种编译器在你加上'use strict';那一刻就会在代码里标红）

函数

定义函数

// 定义方式1
function 方法名(参数) {
  方法体
  [return ...;]
}

// 定义方式2
var 方法名 = function(参数) {
  方法体
  [return ...;]
}

注意：若没有或没有执行 return 语句，函数执行完也会返回结果，返回 undefined

调用函数

无论是用哪种方法定义的函数，都是用 方法名(参数) 来调用

参数问题：JavaScript 的函数调用可以无视方法所规定的传参格式。你可以不传递参数（传入 undefined），也可以传递超过规定量的参数（只取前 n 个）。这么做都不会报错

规避参数问题（不传参）：

可以使用 typeof 判断类型并手动抛出异常的方式

if (typeof x !== 'number') {
  throw 'Not a number';
}

规避参数问题（超量传参）：

可以使用 argument 关键字，该关键字用于接收所有传入的参数，将他们变为一个类似数组的东西（但它不是真正的数组）。它有 length 属性，因此就可以遍历，下标取值：

for (let i = 0; i < arguments.length; i++) {
  console.log(arguments[i]);
}

（ES6 新）获取所有多余参数：

在定义的参数后面在多加一个可变参数 ...rest（这个 rest 可以换成其他名字，它只是个参数名，不是关键字），它会接收所有除定义以外的所有参数，并将它们组成一个真正的数组：

function test(num1, num2, ...args) {
  console.log(args);
}
