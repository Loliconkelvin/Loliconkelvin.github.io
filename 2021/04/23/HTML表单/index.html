HTML表单

发表于 2021-04-23 更新于 2021-05-17 分类于 HTML

重头戏！！！表单

表单父标签（form）

  • 以下所有标签必须包含在form标签内
    <form method="" action=""></form>

method属性（get/post）

  • 定义表单的提交方式
    • get：可以在跳转的url中看到提交的信息，不安全，高效
    • post：比较安全，并且可以传输大文件

action属性

  • 用于定义表单的提交位置
    • 可以是一个网址
    • 可以是一个处理请求的地址

表单子标签——input标签

input标签

<input type=""/>

type属性（必填）

  • type属性定义文本框的类型（作用）
type元素的值 文本框作用
text（默认） 文本框
password 密码框
radio 单选框（圆形）
checkbox 复选框（方形）
hidden
file 用来传文件的框
button 普通按钮
image 图片按钮
submit 表单提交按钮
reset 表单重制按钮

name属性

  • 用于表单的提交

value属性

  • 表示元素的（初始）值（默认值）
  • type="radio"和type="checkbox"时必须指定一个value值，此时表示选择该单选/复选框返回的值

size属性

  • 指定表单的初始宽度
  • type="text"和type="password"时，以字符数为单位
  • 其他类型时以像素为单位

maxlength属性

  • type="text"和type="password"时，指定输入的最大字符数

checked属性

  • type="radio"和type="checkbox"时，指定按钮是否被选中

文本框与密码框

<input type="text"/>
<input type="password"/>
  • 文本框可以使用value属性指定初始值
  • 密码框也行……不会吧不会吧，不会有人给密码框一个初始值吧

单选框（radio）

<input type="radio" value="" name="" id=""/>
  • 单选框必须指定如下属性：
    1. type=”radio”
    2. value: 表示单选框的返回值
    3. name: 表示单选框从属的组，用于表单提交
    4. id: 若使用label标签，该项必填，作为该单选框唯一的标识
  • 同一组单选框只能同时选择一个，因此在N个值需要单选时，他们的name值必须要相同

复选框（checkbox）

<input type="checkbox" value="" name="" id=""/>
  • 单选框必须指定如下属性：
    1. type=”checkbox”
    2. value: 表示复选框的返回值
    3. name: 表示复选框从属的组，用于表单提交
    4. id: 若使用label标签，该项必填，作为该复选框唯一的标识
  • 同一组复选框的name值最好相同，相同组的复选框会以数组的形式返回

checked属性

  • 单选框与复选框可以使用checked属性对其进行默认选中

label标签

<label for=""></label>
  • 通常在单选框和复选框的input标签前加入label标签
  • 效果：当鼠标指向与单选框/复选框相关的显示的值时，同样可以选到单选框/复选框，而不用刻意去点单选框/复选框

for属性

  • 表示这个label属性是与哪个单选框/复选框相关
  • for的值必须是一个单选框/复选框的id值

label标签中包含的值

  • 表示要显示的值

文件域（file）

<input type="file" name=""/>
  • 文件域必须包含如下属性：
    1. type=”file”
    2. name: 用于表单提交，一般定义为files

普通按钮

<input type="button" name="" value=""/>
  • 普通按钮必须包含如下属性：
    1. type=”button”
    2. name: 用于表单提交
    3. value: 表示按钮上显示的文本

图片按钮

<input type="image" src=""/>
  • 图片按钮必须包含如下属性：
    1. type=”image”
    2. src: 表示图片路径

表单提交按钮与表单重置按钮

<input type="submit"/>
<input type="reset"/>
  • 表单提交按钮可以使用value属性改变其显示的值
    • 表单提交按钮默认显示提交
    • 表单重制按钮默认显示重置

表单子标签——select标签

下拉选项框

<select name="">
    <option value=""></option>
    <option value=""></option>
</select>

name属性

  • 用于表单提交

option标签

  • option标签必须包含在select标签内，表示下拉框的可选选项

value值

  • 选择该选项时的返回值

option标签内包含的内容

  • 表示该选项显示的文本

selected值（应用于option标签）

  • 当一个option标签内有selected值时，该选项默认选中

表单子标签——textarea标签

文本域

<textarea name="" cols="" rows=""></textarea>
  • 不可以用value指定初值
  • 但可以在标签内随便写点什么指定初值

name值

  • 用于表单提交

cols属性与rows属性

  • 用于指定文本域的初始显示大小，文本域的显示大小可以在网页端调节

带验证或特殊功能的表单子标签（只介绍选填属性）

邮箱验证

<input type="email" name=""/>
  • 验证方式：
    • 用户所输入的值必须要有@
    • @后面要有值

URL验证

<input type="url" name=""/>

数字验证

<input type="number" name="" min="" max="" minlength="" maxlength="" step=""/>

max属性与min属性

  • 规定用户输入的数字的最大和最小值

step属性

  • 加入step属性可以使输入框右边多一个递增、递减箭头
  • step的值指定箭头按一次增加/减少多少输入框内的值

滑块标签

<input type="range" name="" min="" max="" step=""/>

搜索框

<input type="search" name=""/>
  • 搜索框会在文本框的基础上在框右边加一个可以一键清除输入内容的×

拓展一些属性

隐藏域

  • 在表单子标签内加入 hidden 属性
  • 可以用来传递一些不想让用户修改的默认值

禁用

  • 在表单子标签内加入 disabled 属性
  • 与只读的区别：
    1. 对所有表单元素有效
    2. 使用GET或POST方法提交表单时，当某表单被禁用，其值不会被提交
    3. 用户不能获取表单的焦点，更不能选中其值

只读

  • 在表单子标签内加入 readonly 属性
  • 与禁用的区别：
    1. readonly 只对text和textarea有效
    2. 其值会作为form的一项进行提交
    3. 用户可以获取表单焦点，可以选中其值
