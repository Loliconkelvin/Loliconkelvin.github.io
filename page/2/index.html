注释

行注释

  1. 说明：
    行注释对一行进行注释，适合注释字数少的时候使用
  2. 快速输入：
    无
  3. 代码规范：
    • 方法内部单行注释，在被注释语句上方另起一行，使用//注释。方法内部 多行注释使用/* */注释，注意与代码对齐。

段落注释

  1. 说明：
    段落注释允许写多行注释，适合注释字数多的时候使
  2. 快速输入：
    /* + enter
  3. 代码规范：
    无

方法注释（Javadoc）

  1. 说明：
    详见：https://blog.csdn.net/vbirdbest/article/details/80296136
  2. 快速输入：
    /** + enter
  3. 代码规范：
    • 类、类属性、类方法的注释必须要使用Javadoc规范，禁止使用行注释
    • 所有的抽象方法（包括接口中的方法）必须要用 Javadoc 注释,除了返回值、参数、异常说明外，还必须指出该方法做什么事情，实现什么功能。

标识符和关键字

标识符（含变量、方法、类命名风格）

  1. 说明：
    Java的所有组成部分都需要名字。类名，变量名和方法名都被称为标识符
  2. 代码规范：
    • 代码中标识符的命名均不能以下划线或美元符号开始，也不能以下划线和美元符号结束。
    • 禁止拼音英文混合命名和纯中文命名
    • 类名使用单词首字母大写形式。（如：XmlService）
    • 方法名、参数名、成员变量、局部变量使用驼峰命名。
    • 常量命名字母全部大写，单词间用下划线隔开。

      关键字

  3. 说明：
    Java关键字是电脑语言里事先定义的，有特别意义的标识符，有时又叫保留字，还有特别意义的变量。 Java的关键字对Java的编译器有特殊的意义，他们用来表示一种数据类型，或者表示程序的结构等，关键字不能用作变量名、方法名、类名、包名和参数。

变量类型优先级

byte, short, char -> int -> long -> float -> double

强制转换

  1. 数据类型容量从高转到低需要强制类型转换

    • 禁止对布尔值进行转换
    • 注意内存溢出（浮点数转换为整数时、大数字存储）与精度问题（转换为浮点数时）

变量

  1. 说明
  • 变量即变化的量，可以改变的量
  • 变量分为三种类型：
    1. 局部变量
      • 即写在函数里面的变量，其作用域为函数范围，随函数出现而出现，随函数消失而消失
    2. 实例变量（面向对象）
      • 写在类里面，指在一个类里面 某个或多个对象（实例）拥有但值不一定的变量（属性） ：
      • 在“人”类里面，有sex实例变量，我为man，我的wife为woman。
    3. 类变量（面向对象）
      • 写在类里面，指整个类的属性，需要使用类.类变量的方式提取或赋值。
    • 实例变量和类变量的默认值：
      数据类型 默认值
      整型 0
      浮点型 0.0
      布尔类型 false
      其他类型 null
  1. 代码规范
    • 局部变量命名为首字母小写的驼峰原则v

常量（final）

  1. 说明：

    • 常量声明时即赋值，是一旦声明就不可以改变的量

  2. 代码规范：

    • 常量名必须全大写
    • 多单词使用下划线分隔开
