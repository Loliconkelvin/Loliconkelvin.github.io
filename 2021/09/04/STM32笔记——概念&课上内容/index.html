理论点灯大师（抄PPT）

嵌入式系统结构

嵌入式系统由

  • 微处理器
  • 外围硬件设备
  • 嵌入式操作系统（可选）
  • 用户的应用软件系统

组成

存储限定符 volatile（待求证）

  • 编译器优化工作的一个假设
  • 在多任务共享变量、中断与任务共享变量时，必须使用volatile限定符，强制只读取内存中的数据
void main(void){
    volatile int *p_status;
    int *p_data;

    p_status = (int*) 0x4001;
    p_data = (int*) 0x4000;
    do {

    }while((*p_status & 0x01) == 0);
    ……
}

ARM处理器分类——按应用特征分类

  • ARM A系列 （application）

    偏应用型、高性能、合理功耗

  • ARM R系列 （real-time）

    注重实时性、较低功耗

  • ARM M系列 （micro）

    注重低功耗、低成本

GPIO（Kinetis KL25）（基于寄存器）

  • 点灯总攻略
    1. 配置时钟
    2. IO口模式选择
    3. PDDR（port degital direction register）IO方向选择（输入？输出？）
    4. PDIR（port degital input reg）输入控制
      或
      PDOR（port degital output reg）输出控制
  1. 时钟配置（SIM）

  • SIM_SCGC5的9-13位分别控制A-E五组引脚的时钟开关，为它们置一以开启他们的时钟
/*使用或运算将某一位置一，左移操作控制将哪一位置一
语义清除，比直接写16进制数更容易理解
如果是需要将两位置一的话，或许可以将0x03左移？
*/
SIM_SCGC5 |= (1 << n);
  1. IO口模式选择（MUX）
  • MUX寄存器（8-10位）控制指定引脚的工作模式（作为IO口还是用作其他功能？）
  • 因为一句只能控制一个引脚，所以往往需要写很多句
    /*
x为引脚组编号（A-E）
n为该组编号的引脚号（0-15）
*/

/*
将指定组的指定引脚控制为GPIO模式
即将10-8位置为001，那么11-0位就是
x001 xxxx xxxx。
只需要将原值与0001 0000 0000求与即可
16进制就是0x100
*/
PORTx_PCRn |= 0x100;
  1. 方向选择（PDDR）
  • PDDR寄存器的0-31号引脚分别控制指定组的32个引脚的方向
    置0则指定引脚为输入模式
    置1则指定引脚为输出模式
/*x为引脚组编号（A-E）
将该组0-7号引脚设置为输出模式
*/
GPIOx_PDDR |= 0xFF;
  1. 输出（PDOR）
  • 当引脚为GPIO输出模式时，PDOR控制指定组的32个引脚的逻辑输出电平
/*x为引脚组编号（A-E）
设置该组0-7号引脚输出低电平
*/
GPIOx_GPIO &= (~0xFF);
  1. 点灯总程序
  • 将D组0-7连接的LED灯点亮
int mian(void){
    //开启时钟
    SIM_SCGC5 |= (1 << 12);

    //配置为GPIO模式
    PORTD_PCR0 |= 0x100;
    PORTD_PCR1 |= 0x100;
    PORTD_PCR2 |= 0x100;
    PORTD_PCR3 |= 0x100;
    PORTD_PCR4 |= 0x100;
    PORTD_PCR5 |= 0x100;
    PORTD_PCR6 |= 0x100;
    PORTD_PCR7 |= 0x100;
    
    //配置为输出模式
    GPIOx_PDDR |= 0xFF;

    //配置为输出低电平
    GPIOx_GPIO &= (~0xFF);

    while(1);
}

嵌入式系统中的C语言

数据类型

  • 单片机编程独有代码规范：
    • 尽量使用满足要求的最小数据类型
    • 避免浮点数
    • 在不同系统，不同编译环境给每个数据类型重命名一个宽度值作为标记（单片机独有——移植性）
//For 8 Bit Machine
typedef unsigned char uint8_t;//8位 1字节数据类型
typedef int int16_t;//16位 2字节数据类型
typedef unsigned long uint32_t;//32位 4字节数据类型

//For 32 Bit Machine
typedef unsigned char uint8_t;//8位 1字节数据类型
typedef short int16_t;//16位 2字节数据类型
typedef unsigned int uint32_t;//32位 4字节数据类型

//注意，上面两段根据机器位数取一段使用
  • 在定义变量时使用重命名好的数据类型名（数据类型没有颜色，莫得灵魂）

点灯大师（STMF103CXXXX）（基于寄存器）

  • 还是那四步
    1. 开启时钟
    2. 配置IO口工作模式
    3. 配置IO口方向
    4. 写输出电平
  1. 开启时钟（RCC_APB2ENR）
  • 通过配置RCC_APB2ENR的第2-6位（分别对应IO口组A-G）以开启对应组的时钟
  1. 配置IO口方向与工作方式（GPIO_CPL/GPIO_CPH）
  • F103的IO口工作方向和工作方式是一起配置的，由于一组IO口有16个，因此使用两个寄存器，端口配置低寄存器（GPIOx_CPL） 和端口配置高寄存器（GPIOx_CPH） 分别用于控制 低8位IO口 和 高8位IO口

  • 从属于MODEy位用于控制对应编号的IO口方向

  • 从属于CNFy的用于控制对应编号的IO口工作模式
  1. 写输出内容（GPIO_ODR）
  • 配置GPIO_ODR的0-15位以分别配置16个IO口的输出电平状态

可读可写？
当IO口为输出模式时，GPIO_ODR的0-15位可以写数据，也可以读一下写了什么数据，但当IO口为输入模式时,只可读取，写入无效

  1. 范例程序——点亮PA5口LED
    int main(void){
    
    //配置时钟
    RCC_APB2ENR |= (0x01 << 2);

    //配置方向和模式(推挽输出 50MHz)

    //配置CNF5 = 00
    GPIO_CRL &= ~(0x0C << (5*4));

    //配置MODE5 = 11
    GPIO_CRL |= (0x03 << (5*4));

    //写输出内容（与置零 或置一）
    GPIOA_ODR &= ~(0x01 << 5);

    while(1){

    }
}
  • 注意点：
    1. 配置GPIO_CRL或GPIO_CRH寄存器时要注意它是四个位一组控制一个IO口，因此在按位移动的时候需要移动 IO口编号 * 4 个位数
    2. 用于置零的取反要最后进行
          //配置CNF5 = 00
    GPIO_CRL &= ~(0x0C << (5*4));

    /*
    右移：
        0000 1100 0000 0000…… 右移至目标位置
    取反：
        1111 0011 1111 1111…… 将CNF5置为00
    */

若改为：

    //配置CNF5 = 00
    GPIO_CRL &= (~0x0C << (5*4));

    /*
    取反：
        1111 0011
    右移：（右边补0！！！）
        1111 0011 0000 0000 ……
不仅CNF5被置为00，低位的MODE4 CNF4 …… MODE0 CNF0 也被置零
    */
    3. 与置零，或置一

通讯

通讯的类型

  • 串行/并行
    串行一条线：一次发一位
    并行多条线：一次发多位


  • 同步/异步
    同步：使用同一个时钟
    异步：分别使用各自时钟


  • 点对点/总线
    点对点：只能存在两部通讯设备，不需要设备地址
    总线：多设备，需要设备地址


  • 半双工/全双工
    半双工：线路允许双向传送资料，但同一时刻智能单向传输
    全双工：线路允许同时双向传送资料


  • 主从/对等
    主从：仅主机可以开启通讯，客机需要请求通讯（如：你和迅雷服务器）
    对等：众生平等，谁都可以开启通讯（如：BT传输）


  • 单端/差分
    单端：一个信号一条线
    差分：一个信号使用两条线传输（两线相位相同振幅相反），接收端比较两个电压的电压差来判断发送的逻辑状态，高度免疫外界电磁干扰

UART（通用异步收发器）

  • 书上的USART为 通用同步异步收发器

  • 使用UART通讯，除共地引脚之外，还需要至少两个引脚（RX TX）


  • 空闲状态、开始位和停止位

    1. 当空闲状态时，TX总为 高电平
    2. 开始位固定为 低电平
    3. 停止位固定为 高电平，且停止位占用位数可调

  • 数据的传输

    • 数据从最低位（LSB）开始发送，最后发送最高位（MSB）

  • 波特率（b/s）

    • 看单位就知道，指的是每秒传输的数据位

  • 校验位

    • 奇校验
      奇校验会把数据位中1的个数变为奇数，而不是判断是否为奇数
    • 偶校验
      奇校验会把数据位中1的个数变为偶数，而不是判断是否为偶数

  • 通讯协议

    • 通讯协议的写法为：
      波特率，数据位数 有无校验位 停止位占用位数
    • 如PPT写法：9600,8N1
      代表波特率9600，八位数据位，无校验位，停止位占用一位
有无校验位 写法 对应
无校验位 N null
奇校验 O odd
偶校验 E even

SPI（全双工同步串行通讯总线）

  • 使用SPI通讯，需要4个引脚

SS（CS）片选信号

  • 固定空闲状态为高电平，当变为低电平时即开始传输数据

SCLK时钟信号

  • 由主设备产生，有两个参数可调，对应四种模式

  • CPOL空闲电平对传输协议没有重大影响，只是决定空闲状态的电平，无论空闲状态是什么电平，开始传输的时候都是高低电平交替

SPI时钟模式 CPOL空闲电平 CPHA采样时刻
空闲低电平 第一个时钟边沿采样 0 0
空闲低电平 第二个时钟边沿采样 0 1
空闲高电平 第一个时钟边沿采样 1 0
空闲高电平 第二个时钟边沿采样 1 1
  • 当使用第一时钟边沿采样，当发送下一次数据时，SS必须先变高电平
  • 当使用第二时钟边沿采样，当发送下一次数据时，SS可以不跳变
